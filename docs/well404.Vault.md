# well404.Vault

每位玩家一个**私人仓库**:把背包里的物品存进去、需要时再取出来。物品**完整保真**保存(耐久、品质、
枪械配件、弹匣/弹药箱内的弹药等),取出原样还原。容量按**背包格子数**计。

## 依赖

- **无硬依赖**,可独立运行(仅用指令)。
- **可选**:[`well404.WebPanel`](well404.WebPanel.md) → 提供玩家网页仓库与容量的可视化设置。

```bash
openmod install well404.Vault
openmod reload
```

## 命令

> OpenMod 按命令路径派生权限,需在权限角色里授予。

| 命令 | 语法 | 权限 | 说明 |
| --- | --- | --- | --- |
| `/vault`（`/storage`） | | `well404.Vault:commands.vault` | 显示用法 |
| `/vault store`（`deposit`） | `<物品ID> [数量]` | `well404.Vault:commands.vault.store` | 把背包里该物品存入仓库 |
| `/vault take`（`withdraw`） | `<物品ID> [数量]` | `well404.Vault:commands.vault.take` | 把仓库里该物品取回背包 |
| `/vault list`（`info`） | | `well404.Vault:commands.vault.list` | 列出仓库内容与占用 |

- **按游戏物品 ID 操作**:`/vault store 363 2` 存入 2 个物品 363;`/vault take 363` 取回 1 个。
- 取出时若背包放不下,物品会**掉落在玩家脚下**(不会丢失)。
- `store` 一次最多存入背包中匹配该 ID 的若干个;若仓库容量不足会**存到放不下为止**并提示已满。

## 容量与计费(按格子)

- 每个存入的物品占用其**网格尺寸**(`size_x × size_y`):例如 1×1 的小物品占 1 格、2×2 的弹药箱占 4 格。
- **物品内部的堆叠数 / 弹药数不计入占用**:一盒装着 39 发子弹的弹药箱,仍只按它的格子尺寸占用,
  **不会**被算成 39 个物品。读不到某物品尺寸时,该物品按 1 格计。

### 每位玩家的容量(基础 / 权限档位 / 单独覆盖)

容量是**按玩家**计算的,优先级从高到低:

1. **单独覆盖**:为某位玩家(按 Steam ID)单独设定的容量,**优先于一切**。在 Web 管理面板
   「仓库 → 单独玩家容量」里增删改;持久化在 `vault.data.yaml` 的 `overrides`。
2. **权限档位**(`tiers`):玩家持有某权限即获得对应容量;玩家取「基础容量与其拥有的各档位」中的
   **最大值**(所以 VIP 仓库更大)。在 config / 面板里配置 `权限=容量`。
3. **基础容量**(`maxSlots`,默认 200):所有人至少有的容量。

## 完整保真

存入时记录物品的 **id、堆叠数(amount)、品质(quality)、原始状态字节(state)**;取出时用这些原样重建,
因此**枪械的配件、弹匣里的弹药、弹药箱内的余弹、物品耐久**等都会保留。仓库数据以
`vault.data.yaml` 持久化于插件数据目录(按 Steam ID 分组,状态以 base64 存储)。

## 配置（`config.yaml`）

```yaml
# 基础容量(背包格子数);每个物品按其网格尺寸(宽×高)占用;内部堆叠/弹药数不计入。
maxSlots: 200

# 按权限的容量档位;玩家取「基础容量与其拥有的各档位」中的最大值。空 = 无档位。
tiers: {}
# 示例：
# tiers:
#   "well404.vault.size.vip": 400
#   "well404.vault.size.mvp": 600
```

装了 WebPanel 时,这些值也可在 Web 界面编辑(管理面板「仓库」模块:基础容量 + 档位;以及
「单独玩家容量」按 Steam ID 单独覆盖)。单独覆盖是运行时数据,存于 `vault.data.yaml`。

## Web 面板(可选)

装了 [`well404.WebPanel`](well404.WebPanel.md) 后:

- **管理面板**:注册「仓库」模块(图标 🧳):
  - **设置**:基础容量 `maxSlots` + 权限档位 `tiers`(格式 `权限=容量, …`),页尾统一保存;
  - **单独玩家容量**:集合式 CRUD,为某个 Steam ID 单独设定容量(优先于基础/档位)。
- **玩家网页仓库**:注册一个面向玩家的「仓库」菜单(玩家面 `/p`,图标 🧳)。玩家 `/menu vault` 收到链接,
  在浏览器里看到:
  - 顶部容量徽章「仓库:X / Y 格」(Y 为该玩家的实际容量);
  - **背包**分区:当前背包里的物品(按 ID 分组),每项一个「存入」按钮——点按弹出数量,**默认填该物品的
    全部数量**(直接确认即批量全存);
  - **仓库**分区:已存物品(显示数量与占用格数),「取出」按钮(数量默认填全部),多于一件时还有「详情」;
  - **详情(二级子菜单)**:点「详情」进入该物品的逐件列表,**每一件单独显示其特征**——数量/弹药(`×N`)、
    耐久(`耐久 X%`)、是否带内部配置(`🔧`,如枪械配件);每件一个「取出」按钮,可精确取出**指定的那一件**
    (例如同是 12 Gauge Shells,5 发的那盒和 8 发的那盒分开列出,各自取出),「← 返回」回到上一级。
  - 存取须玩家在线(操作其实时背包);**仅查看**仓库内容则离线也可。该页面完全由通用的玩家菜单模型驱动
    (分区/列表/徽章/按钮),WebPanel 宿主不含仓库专属逻辑。
