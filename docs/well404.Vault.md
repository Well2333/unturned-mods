# well404.Vault

每位玩家一个**私人仓库**:把背包里的物品存进去、需要时再取出来。物品**完整保真**保存(耐久、品质、
枪械配件、弹匣/弹药箱内的弹药等),取出原样还原。容量按**背包格子数**计。

## 依赖

- **无硬依赖**,可独立运行(仅用指令)。
- **可选**:[`well404.WebPanel`](well404.WebPanel.md) → 提供玩家网页仓库与容量的可视化设置。

```bash
openmod install well404.Vault
openmod reload
```

## 命令

> OpenMod 按命令路径派生权限,需在权限角色里授予。

| 命令 | 语法 | 权限 | 说明 |
| --- | --- | --- | --- |
| `/vault`（`/storage`） | | `well404.Vault:commands.vault` | 显示用法 |
| `/vault store`（`deposit`） | `<物品ID> [数量]` | `well404.Vault:commands.vault.store` | 把背包里该物品存入仓库 |
| `/vault take`（`withdraw`） | `<物品ID> [数量]` | `well404.Vault:commands.vault.take` | 把仓库里该物品取回背包 |
| `/vault list`（`info`） | | `well404.Vault:commands.vault.list` | 列出仓库内容与占用 |

- **按游戏物品 ID 操作**:`/vault store 363 2` 存入 2 个物品 363;`/vault take 363` 取回 1 个。
- 取出时若背包放不下,物品会**掉落在玩家脚下**(不会丢失)。
- `store` 一次最多存入背包中匹配该 ID 的若干个;若仓库容量不足会**存到放不下为止**并提示已满。

## 容量与计费(按格子)

- 仓库总容量是 `maxSlots` 个**背包格子**(默认 200)。
- 每个存入的物品占用其**网格尺寸**(`size_x × size_y`):例如 1×1 的小物品占 1 格、2×2 的弹药箱占 4 格。
- **物品内部的堆叠数 / 弹药数不计入占用**:一盒装着 39 发子弹的弹药箱,仍只按它的格子尺寸占用,
  **不会**被算成 39 个物品。
- 读不到某物品尺寸时,该物品按 1 格计。

## 完整保真

存入时记录物品的 **id、堆叠数(amount)、品质(quality)、原始状态字节(state)**;取出时用这些原样重建,
因此**枪械的配件、弹匣里的弹药、弹药箱内的余弹、物品耐久**等都会保留。仓库数据以
`vault.data.yaml` 持久化于插件数据目录(按 Steam ID 分组,状态以 base64 存储)。

## 配置（`config.yaml`）

```yaml
# 仓库总容量(背包格子数)。每个物品按其网格尺寸(宽×高)占用;内部堆叠/弹药数不计入。
maxSlots: 200
```

装了 WebPanel 时,这个值也可在 Web 界面编辑(管理面板「仓库」模块)。

## Web 面板(可选)

装了 [`well404.WebPanel`](well404.WebPanel.md) 后:

- **管理面板**:注册「仓库」模块(图标 🧳),一个设置项「容量(格子数)」即 `maxSlots`,页尾统一保存。
- **玩家网页仓库**:注册一个面向玩家的「仓库」菜单(玩家面 `/p`,图标 🧳)。玩家 `/menu vault` 收到链接,
  在浏览器里看到:
  - 顶部容量徽章「仓库:X / Y 格」;
  - **背包**分区:当前背包里的物品(按 ID 分组,显示数量),每项一个「存入」按钮(可输入数量);
  - **仓库**分区:已存物品(显示数量与占用格数),每项一个「取出」按钮(可输入数量)。
  - 玩家须在线(存取要操作其实时背包)。该页面完全由通用的玩家菜单模型驱动(分区/列表/徽章/按钮),
    WebPanel 宿主不含仓库专属逻辑。
