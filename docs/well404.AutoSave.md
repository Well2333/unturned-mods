# well404.AutoSave

**定时保存游戏**并**周期性地把存档高压缩备份**。按 **crontab 墙钟**触发保存（默认每 10 分钟），
有玩家时每保存若干次顺带做一次 **LZMA 实体压缩**备份；无人在线时在一次正常备份后自动切换到长周期归档；备份目录、排除规则、保留上限均可配置。

> **没有任何游戏内指令**：全部通过 `config.yaml` 配置；装了
> [`well404.WebPanel`](well404.WebPanel.md) 时可在管理面板里可视化配置、查看/删除备份、立即备份。

## 依赖

- **无硬依赖**，可独立运行。
- 随包自动安装两个纯托管库：`Cronos`（cron 解析）、`SharpCompress`（LZMA 压缩）。
- **可选**：[`well404.WebPanel`](well404.WebPanel.md) → 提供可视化配置与备份管理。

```bash
openmod install well404.AutoSave
```

安装或更新插件二进制后必须完整重启服务器；`openmod reload` 仅用于同版本配置重载。

## 工作方式

1. **定时保存**：按 `schedule.cron`（标准 5 段 cron）在**墙钟边界**触发 `SaveManager.save()`。
   例如默认 `*/10 * * * *` 在每个 `:00/:10/:20…` 保存——与 crontab 一致，**不随服务器启动时刻计时**。
   时区由 `schedule.timeZone` 决定（留空=服务器本地时区）。
2. **活跃期备份**：维护一个**持久化**的保存计数（存于插件数据目录 `save-count.txt`，重启后延续）。
   有玩家时，每当计数是 `backup.everyNSaves` 的整数倍，就在那次保存后立刻做一次备份。默认 `6` →
   配合默认 cron 即「每小时一次备份」。
3. **空服限频**：最后一名玩家离线后不立刻改变自动保存；下一次本来应当发生的常规备份照常完成，
   成功后才切换为 `idleBackup.intervalHours`（默认 24 小时）一次。首名玩家重新上线（0→1）时只
   恢复活跃期的 `backup.everyNSaves` 正常节奏，不会在连接流程中立即保存或压缩归档。
4. **自动保存不降频**：无论在线人数与空服模式如何，`SaveManager.save()` 始终按 `schedule.cron` 执行；
   变化的只有压缩归档频率。
5. **打包**：把存档目录按排除规则流式打进一个 `.tar.lz`（tar + LZMA 实体压缩，跨文件整体压缩，
   体积最省；7-Zip / lzip / SharpCompress 均可解开）。先写 `*.tmp` 再原子改名，避免半成品。
6. **保留**：每次新备份后，若备份数超过 `retention.maxCount` 或总体积超过
   `retention.maxTotalSizeMB`（任一），删除**最旧**的若干个，直到两个上限都满足；**永远至少保留最新一份**。

保存在主线程同步完成（保证文件落盘一致），随后的打包/删除在后台线程进行，不卡服。
同一时刻只跑一个备份：普通计划/手动请求与已有备份重叠时会被**跳过并告警**（保存照常）。
玩家连接事件不会等待备份，避免压缩任务延迟连接握手。

## 备份什么 / 不备份什么

默认**备份整个** `Servers/<服务器id>/`，再**减去**一组「可下载 / 可再生」的排除项。未在排除列表里的
新目录默认**包含**（宁可多备，不丢数据）。备份目录自身始终被自动排除（避免备份套备份）。

| 默认会备份（必要存档） | 默认排除（可下载 / 可再生） |
| --- | --- |
| `Level/`（世界：建筑、载具、地形改动） | `Workshop/**`、`Steam/**`（创意工坊下载、Steam 缓存） |
| `Players/`（玩家存档：背包、位置、技能） | `OpenMod/packages/**`（OpenMod 的 NuGet 包缓存，可重下） |
| `Server/`（管理员 / 封禁 / 白名单） | `Bundles/**`（资源缓存） |
| `Config.txt`（服务器配置） | `**/logs/**`、`**/Logs/**`（日志） |
| `OpenMod/` 下角色 / 用户 / 各插件配置与数据 | `**/*~`、`**/*.bak`（临时 / 备份残留） |

排除规则是相对存档根的 glob：`*` 不跨目录、`**` 跨目录（含零段）、`?` 匹配单个非分隔符；**大小写不敏感**。

## 配置（`config.yaml`）

```yaml
schedule:
  cron: "*/10 * * * *"   # 标准 5 段 cron（分 时 日 月 周）；默认每 10 分钟，按墙钟对齐
  timeZone: ""            # 空=服务器本地时区；否则 IANA(如 Asia/Shanghai)或 Windows 时区 id
backup:
  enabled: true          # 关掉则仍定时保存，只是不备份
  everyNSaves: 6          # 每 N 次保存备份一次；<=0 关闭备份
  directory: ""           # 空=<安装目录>/Backups/<服务器id>；绝对路径原样用，相对路径相对安装根
  excludePatterns:        # 见上「不备份什么」
    - "Workshop/**"
    - "Steam/**"
    - "Bundles/**"
    - "OpenMod/packages/**"
    - "**/logs/**"
    - "**/Logs/**"
    - "**/*~"
    - "**/*.bak"
idleBackup:
  enabled: true            # 无人在线时降低归档频率；自动保存频率不变
  intervalHours: 24        # 首次常规空服备份成功后，长周期归档间隔（1–8760 小时）
retention:
  maxCount: 0             # 最大备份数，0=不限
  maxTotalSizeMB: 0       # 最大总体积(MB)，0=不限
```

## Web 管理面板（装了 `well404.WebPanel` 时）

面板里出现 **Auto Save** 模块，包含：

- **计划与备份**（设置组）：上述所有配置项都能在 WebUI 编辑，保存即重写 `config.yaml` 并热重启调度；
  包括空服限频开关与空服备份间隔。
- **立即备份**（按钮）：立刻保存并写一次备份（即使关闭了定时备份也会执行），返回备份名/大小/文件数。
- **备份列表**（表格）：列出现有备份（名称 / 大小 / 时间，最新在前），每行可**删除**。

界面与提示**中英双语**，按浏览器选择的语言显示。

## 恢复备份

备份是标准的 `.tar.lz`（tar + LZMA）。停服后用 7-Zip / `lzip`/`tar` 解压到对应 `Servers/<id>/`
目录覆盖即可（解压时注意只覆盖你要回滚的内容）。本插件只负责**创建**备份，不在游戏内做自动恢复。

## 还原 / 卸载

卸载插件即停止定时保存与备份；已生成的备份文件不受影响，保留在备份目录里。
