# 各插件用法文档 + 本地构建/部署脚本

## 做了什么
- 新增 `docs/`：
  - `docs/README.md`：文档索引、安装（`openmod install` 或手动 dll）、本地构建与调试
    （`scripts/build.sh` 用法）、命令权限格式说明。
  - `docs/well404.Economy.md`：`config.yaml` 逐项示例、两种货币后端对比、命令与权限串、
    击杀奖励、**其他插件注入 `IEconomyProvider`** 的对接代码示例。
  - `docs/well404.Shop.md`：`config.yaml` 示例（单物品 + 组合包）、命令与权限、按权限组
    折扣的完整配置步骤（含 `permissionRoles.yaml` 授权示例）。
  - 命令权限串按 OpenMod `CommandPermissionBuilder` 实测确认：
    `<ComponentId>:commands.<父.子>`（如 `well404.Economy:commands.eco.give`）。
- 新增 `scripts/build.sh`：本地 build / `--test` / `--pack` / `--clean`，以及
  `--deploy <dir>`：`dotnet publish` 单插件后**只复制「插件 dll + 该插件非 OpenMod 的
  PackageReference 依赖」**（如 well404.Economy 带 `LiteDB.dll`，well404.Shop 仅插件 dll）
  到本地服务器 plugins 目录，避免把宿主已有的程序集（OpenMod/Unity/UniTask 等）一并
  复制造成版本冲突。
- 删除占位 `docs/.gitkeep`；README 增加 docs 链接与 `build.sh` 用法。

## 为什么
- 用户要求：在 docs 补充每个插件的具体用法与配置示例，并提供本地构建脚本便于自行 debug。

## 影响的文件 / 范围
- 新增 `docs/**`、`scripts/build.sh`；改 `README.md`、
  `memory/guidelines/development-standards.md`；删 `docs/.gitkeep`。

## 注意事项 / 后续
- `--deploy` 的「最小依赖集」依据插件 `.csproj` 的非 OpenMod PackageReference 推导；若某
  第三方依赖还带自己的子依赖（本仓库 LiteDB 无），需另行复制或改用 `openmod install`。
  已在脚本输出与文档中注明。
