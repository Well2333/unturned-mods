# 管理面路径式 token 鉴权 + 可选内置反代(隧道);玩家链接改为在线即有效

## 做了什么
- **管理面改为路径式 token 鉴权**:整个管理面藏在 `/<token>/` 前缀后,token 即鉴权,路径
  不对一律普通 404(不泄露面板是否存在)。`WebPanelHttpServer` 路由按 `/<token>` 前缀剥离后
  再走原 `/api/...` 逻辑;`index.html` 改用相对路径并移除 token 输入框。
- **token 强制存在并持久化**:`web.token` 留空时,插件生成 **12 位大小写字母+数字**(用
  `RandomNumberGenerator`)并**写回 `config.yaml`**(`PersistGeneratedToken`,定向替换空的
  `token:` 行,保留注释),重启保持不变。token **不在 WebUI 中可改**。
- **可选内置反代/隧道**(`web.tunnel`):新增 `ITunnelProvider` 抽象 + 通用
  `ProcessTunnelProvider`:拉起**管理员已安装**的隧道二进制(不内置/不下载),从 stdout/stderr
  正则或本地 JSON API(如 ngrok `127.0.0.1:4040`)解析公网 URL。两种 `type`:
  - `cloudflare`:预置 cloudflared Quick Tunnel 参数与 trycloudflare 正则,**已实测**端到端可用
    (下载 cloudflared 2026.6.0,起本地服务 + 隧道,确认解析出 URL 且公网 200 可达)。
  - `custom`:用户自定义 command/args/urlPattern/apiUrl(如 ngrok)。
  隧道起来后的公网地址经 `PlayerWebSessionManager.SetTunnelBaseUrl` 动态注入玩家链接生成器,
  并用于日志里的管理面地址;插件卸载时关闭隧道。
- **玩家链接有效期规则**:下限抬到 **15 分钟**;超过下限后,只要玩家**仍在线就一直有效,
  下线即失效**(`Validate` 过期后改查 `IUnturnedUserDirectory.GetOnlineUsers`)。
- **CORS**:所有响应加宽松 CORS 头 + 处理 OPTIONS 预检,便于套在任意第三方反代后面。
- **移除 `web.allowInsecurePublic`**(管理面如今恒有 token,该「无鉴权对外」开关已无意义)。
- 版本 WebPanel `0.2.0→0.3.0`。

## 为什么
- 用户要求:大幅提升对外暴露时的安全性。路径式 token(类似 Jupyter/Gradio)比 header 式更
  隐蔽;内置反代让游戏服不暴露真实 IP、不开入站端口、白嫖 HTTPS;隧道不绑死 Cloudflare,
  保留通用接口适配任意反代;玩家链接「在线即有效」更贴合实际游玩时长。

## 影响的文件 / 范围
- 新增:`WebPanel/TunnelProvider.cs`(`ITunnelProvider` + `ProcessTunnelProvider`)。
- 改动:`WebPanelHttpServer.cs`(路径式路由 + CORS + OPTIONS)、`WebPanelPlugin.cs`(token 生成
  /持久化、隧道生命周期、日志)、`WebPanelSettings.cs`(`TunnelSettings`、移除 `AllowInsecurePublic`)、
  `PlayerWebSessionManager.cs`(隧道地址注入、在线门控有效期)、`index.html`(相对路径、去 token 框)、
  `config.yaml`、`.csproj`(版本)。
- 文档:`README.md` + `docs/well404.WebPanel.md` + `docs/README.md` 总表。

## 注意事项 / 后续
- **管理面地址即后台密码**:启动日志打印 `…/<token>/`,务必保管。开隧道时整个端口(含管理面)
  都被反代出去,路径 token 是唯一防线。
- cloudflare 隧道需服务器自行安装 `cloudflared` 并在 PATH;custom 适配 ngrok 需先
  `ngrok config add-authtoken`。
- 反代二进制由插件作为子进程托管,卸载时 kill;若进程中途退出仅记日志(未做自动重启,可作后续)。
