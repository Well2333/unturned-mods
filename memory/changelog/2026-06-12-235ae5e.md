# 玩家面向的 Web 面板（/menu）：商店买卖、钱包转账、便民传送

## 做了什么
- **WebPanel 新增「玩家面」**（与管理员面并行）：
  - 共享契约（`UnturnedMods.Shared/WebPanel/PlayerMenu.cs`）：`IPlayerMenu`、
    `IPlayerMenuRegistry`、`IPlayerWebSessionService` + DTO（`PlayerMenuContext` /
    `PlayerCard` / `PlayerButton` / `PlayerMenuView` / `PlayerActionResult`）。
  - 实现：`PlayerMenuRegistry`（全局单例注册表）、`PlayerWebSessionManager`（短时
    token 会话 + 链接生成 + `Validate()`）。
  - HTTP 路由（`WebPanelHttpServer`）：`GET /p`（玩家 SPA）、`GET /api/p/view`
    （渲染该玩家全部菜单）、`POST /api/p/invoke/{menu}`（执行卡片按钮）。**玩家面用
    一次性会话令牌鉴权（`?t=` / `X-Player-Token`），与管理员 `token` 完全独立、互不通用。**
  - `/menu [tab]` 命令（别名 `/panel`）：用 `IPlayerWebSessionService` 生成链接，主线程
    上 `Player.sendBrowserRequest(message, url)` 推送给玩家。
  - 前端 `player.html`（移动友好，复用管理面主题），作为 EmbeddedResource 嵌入。
  - 新配置项 `web.publicBaseUrl`、`web.playerSessionMinutes`。
- **三个功能插件各注册一个玩家菜单（可选注入 `IPlayerMenuRegistry`，未装面板则跳过）：**
  - Shop `ShopPlayerMenu`：列商品卡片、买/卖（复用 `/buy`+`/sell` 的折扣、扣费、失败
    退款、背包校验逻辑）。
  - Economy `EconomyPlayerMenu`：显示余额；每个在线玩家一张卡片可转账（复用 `/pay` 的
    开关/最低额/税率/扣款入账退款逻辑）。
  - Essentials `EssentialsPlayerMenu`：回家 / 返回死亡点 / 传送到有权限的传送点（复用
    `TeleportService`，含冷却、可选经济费用、静止预热）。
- 版本：WebPanel `0.1.2→0.2.0`、Shop `0.2.2→0.3.0`、Economy `0.2.2→0.3.0`、
  Essentials `0.1.0→0.2.0`。

## 为什么
- 让玩家无需记命令，直接在浏览器里完成买卖、转账、传送等高频操作；服务器侧沿用既有
  命令的成熟逻辑，避免重复实现导致行为分叉。
- 玩家面采用独立短时令牌，杜绝把管理员 token 暴露给玩家。

## 注意事项 / 后续
- 玩家面要可用，必须让玩家浏览器可达本服务：面板绑 `127.0.0.1` 时需配置可达的
  `publicBaseUrl`，否则 `/menu` 会提示不可达。
- `Player.sendBrowserRequest` 在游戏内弹 Steam 叠加层确认框；服务器须能被玩家公网访问。
- Essentials 传送走完整 `TeleportService` 流程，点按钮后玩家可能需在游戏里短暂站立读条；
  具体失败原因仍在游戏内提示。
- 本地已 `dotnet build -c Release` 绿色（0 警告 0 错误）、45 个单测全过；**未推送、未发版**，
  停在本地等待指示。
- 后续可考虑：给玩家面增加 Essentials 礼包领取（`PlayerDataStore` 已有 gift-claim 后端）、
  为玩家菜单/`/menu` 增加可配置的权限节点与展示开关。
