# WebPanel 开发预览（dev-player）+ 玩家商店页面重设计

用户两点诉求：①「商店页面太丑」；②希望「不用上线也能看到玩家面板」——在 WebPanel 里建一个
开发模式专用、绑定到指定账号（Well404）的玩家界面。

## 做了什么

### ① 开发者预览 `web.devPlayer`
- 新增 `DevPlayerSettings`（`enabled`/`steamId`/`displayName`），默认关闭。
- 开启后 `GET /<token>/dev-player` 调 `PlayerWebSessionManager.CreateDevSession`（签发**1 天**长效
  会话，绕过「离线即失效」），再 **302 跳转 `/p?t=<token>`**，直接以该账号进入玩家面板。
- **双重门槛**：既在管理面密钥路径 `/<token>/` 之后，又需 `enabled: true`；关闭时该路径返回普通
  404（无信息泄露）。属玩家身份模拟，生产环境保持关闭。
- 仅预览渲染：买卖/传送等需玩家真正在线的动作仍提示「需要在线」。
- 启动时若开启会打印一条 WRN，附带 `…/dev-player` URL，便于发现。

### ② 玩家商店页面重设计（`player.html`，纯前端）
- 商店标签改为**自适应商品网格**（product tiles）：窄屏单列、宽屏多列；卡片等高、按钮置底。
- 余额由原来的灰色小字改为**徽章药丸**（`.head` 重设计，所有玩家菜单通用）。
- 商店买入按钮在网格内用**绿色**（`--ok`）以突出「购买」；卡片 hover 轻微抬升+阴影。
- 礼包内容的 pill 标签底色调亮（`--panel2`），与卡片背景区分更清晰。
- 实现：渲染时把卡片包进 `.cards` 容器，仅商店标签追加 `.grid`；其余标签维持单列。

## 为什么

- ②是对上一轮（商店显示物品名/分离礼包徽章）之后整体观感的进一步打磨——商店是 N 张小卡片，
  单列堆叠显得空且丑，网格化最贴合「商店」场景。
- ①让玩家面（`/p`，原本必须游戏内 `/menu` 才能拿会话）可在浏览器里直接预览，方便我和用户调试。

## 影响的文件 / 范围

- `WebPanelSettings.cs`（`DevPlayerSettings`）、`config.yaml`（devPlayer 块 + 注释）、
  `PlayerWebSessionManager.cs`（`CreateDevSession`）、`WebPanelHttpServer.cs`（`/dev-player` 路由 +
  `m_DevPlayer` 字段/构造参数）、`WebPanelPlugin.cs`（传 `web.DevPlayer` + 启动 WRN）。
- `player.html`（`.cards`/`.cards.grid` 网格、`.head` 徽章、网格内绿色买入按钮、hover、pill 调色；
  `renderView` 包 `.cards` 容器并对 shop 加 `.grid`）。
- 文档：`docs/well404.WebPanel.md`、`well404.WebPanel/README.md`（devPlayer 配置 + `/dev-player`
  路由）、`docs/well404.Shop.md`（玩家商店商品网格说明）。
