# 新增经济与商店插件，含测试与按插件发布

## 做了什么

### well404.Economy
- 以全局服务实现 OpenMod 标准抽象 `IEconomyProvider`
  （`[ServiceImplementation(Singleton)]`，在插件作用域构造，直接注入插件
  `IConfiguration`/`IOpenModComponent`；与官方 `OpenMod.Economy` 同构）。其他插件
  （签到等）注入该抽象即可发币/扣币。
- **可切换货币后端**（`config.yaml: backend`）：
  - `database`：serverless **LiteDB** 单文件账本（`LiteDbCurrencyBackend`，每次操作
    打开/关闭连接 + 锁串行化，避免文件被长期占用、读改写原子）；账户键
    `ownerType:ownerId`，并写交易流水。在线/离线均可。
  - `experience`：直接用原生 Unturned 经验值（`PlayerSkills.ServerSetExperience`），
    仅在线玩家。
- 命令：`/balance [player]`、`/pay <player> <amount>`（可配最小额/税、失败自动退款）、
  `/eco give|take|set`（管理）。玩家解析支持在线名字/ID，离线 17 位 SteamID（DB 后端）。
- 击杀奖励事件监听：玩家死亡（`UnturnedPlayerDeathEvent.Instigator`）、僵尸/动物用
  **致命伤判定**（在 `*DamagingEvent` 里 `DamageAmount >= Health` 时给击杀者发币，
  普通/Mega 僵尸分别计奖）。非致命命中走廉价早退，不触发经济写入。

### well404.Shop
- `config.yaml` 配置商品目录：单物品（itemId/amount/价格）与**自定义组合包**
  （bundle，多物品内容，可整包售卖）。
- 命令：`/shop`（列表）、`/buy <id> [amount]`、`/sell <id> [amount]`。买入扣币失败抛
  `NotEnoughBalanceException`；发放走 `IItemSpawner.GiveItemAsync`；售卖先校验库存再
  扣物（`DestroyAsync` 按 ItemJar 引用重定位，安全）。买入失败自动退款。
- **按权限组购买折扣**（`DiscountService`）：`discounts.tiers` 映射权限→乘数，取已授予
  的最优（最低）乘数；默认 `enabled: false`。仅作用于买入。
- 仅依赖抽象 `IEconomyProvider`，**不**对 well404.Economy 建 ProjectReference。

### 测试 / CI / 发布
- `tests/well404.Economy.Tests`（xUnit, net8.0, `IsPackable=false`）覆盖**游戏无关纯
  逻辑**：LiteDB 账本（起始余额/增减/透支抛错/转账/持久化）、折扣（取最优档/禁用/
  四舍五入）。12 个用例全过。
- CI：`dotnet test` 已生效，新增 `dotnet pack` 打包校验。
- **按插件发布**：`publish.yml` 解析 Release tag `<PluginId>/v<Version>`，只发布该插件
  并校验版本与 `.csproj` 一致；非该格式则发布全部。新增脚本
  `scripts/plugin-version.sh`、`scripts/release-plugin.sh`。
- 依赖（`Directory.Packages.props`）：新增 `LiteDB 5.0.21`、
  `OpenMod.Extensions.Economy.Abstractions 3.8.10`、测试栈
  （`xunit 2.9.2` / `xunit.runner.visualstudio 2.8.2` / `Microsoft.NET.Test.Sdk 17.11.1`）。

## 为什么
- 用户需求：经济+商店；serverless（类 SQLite）DB；货币来源支持外部调用/击杀/经验值；
  玩家转账；单物品与组合包买卖；按等级（权限组）折扣且默认关闭。并要求多插件 GH
  Action 发布/版本管理与插件测试。

## 注意事项 / 后续
- **未在真实 Unturned 服务器端到端验证**（本环境无游戏运行时）：已做到编译 0 警告 0
  错误、纯逻辑单测全过、API 签名对 3.8.10 程序集反编译核对。游戏内行为（发物品、改
  经验、命令注册、事件触发）需上服务器手测。
- 运行依赖：`well404.Shop` 需场上存在任一 `IEconomyProvider` 实现（well404.Economy 或
  兼容经济插件）；服务器只应启用一个经济提供方。
- 手动 dll 部署时需一并放 `LiteDB.dll`（`openmod install` 会自动拉取）。
- 折扣只作用于买入；如需售卖加成需另行设计。
