# 管理面：商品瓦片显示买/卖价 + 检索快速加商品改为弹窗填价

用户(附截图)要求：①管理面商品瓦片改成「名称 #id + 买价 卖价」那种样式;②点「加入商店」时用弹窗
(两个输入框填买卖价)等更明显的方式提醒管理员设价。

## 做了什么（两个通用能力，商店使用）

- **`WebPanelAction.SummaryFields`**(通用)：`Collection` 的每条记录在瓦片上额外用
  「字段标签: 值」胶囊显示选定字段(标签已服务端本地化)。Shop 的普通商品/礼包集合设
  `["buyPrice","sellPrice"]`,于是每个瓦片直接显示「买价: 100 / 卖价: 40」,不必打开编辑器。
- **`WebActionResult.WithRowAction(..., fields)`**(通用)：表格/检索结果的行内动作可声明一组输入字段,
  点击时**先弹窗收集**再连同行 key 一并提交。Shop 检索的「＋ 加入商店」现在弹出「买价/卖价」两个输入框,
  填好才建商品(不再静默建零价草稿)。

实现：`index.html` 通用渲染——`paint()` 追加 summary 胶囊;`showModal()` 小弹窗收集行内动作字段;
`rowAction()` 有 fields 时先弹窗。服务端 `BuildModulesJson` 输出 `summaryFields`、`BuildResultJson`
输出 `rowActionFields`(抽出通用 `AppendFieldsJson` 复用、标签本地化)。

## 为什么

- 价格直接显示在瓦片省去逐个点开;加商品时强制当场填价,避免「忘了设价导致 0 价不可买卖」。两者都做成
  **通用** WebPanel 能力(非商店专属),符合「宿主不含插件逻辑」。

## 影响的文件 / 范围

- Shared：`WebPanelModule.cs`(`SummaryFields`)、`WebAction.cs`(`RowActionFields` + `WithRowAction` 重载)。
- WebPanel：`WebPanelHttpServer.cs`(序列化 + `AppendFieldsJson`)、`index.html`(summary 胶囊 + 弹窗 +
  `.pill.kv`/`.modal` 样式 + `confirm` 文案)。
- Shop：`ShopWebPanelModule.cs`(两集合 summaryFields;检索 `WithRowAction` 带买卖价字段;`QuickAddItem`
  读取价格、成功文案改「Added to the shop.」)、`WebI18n.cs`。
- 文档：`docs/well404.Shop.md`、`well404.Shop/README.md`、`docs/well404.WebPanel.md`。
