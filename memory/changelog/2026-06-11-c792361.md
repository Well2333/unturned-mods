# 新增 Web 管理面板（well404.WebPanel）并接入经济/商店

## 做了什么
- **新增 `well404.WebPanel`**：BCL `HttpListener` 宿主（零额外依赖）+ 内嵌通用
  schema 驱动 SPA（`index.html`，EmbeddedResource）+ 手写 JSON。`GET /` 出 SPA、
  `GET /api/modules` 出模块元数据、`POST /api/modules/{m}/{a}` 执行动作、
  `GET .../values`（预填）、`GET .../records`（集合列表）、`POST .../delete`（删除）。
- **跨插件注册抽象**放 `UnturnedMods.Shared.WebPanel`：`IWebPanelRegistry`（`[Service]`）+
  描述符（`WebPanelModule/Action/Field/Record/Request/Result`，不含 JSON 类型）。
  WebPanel 以 `[ServiceImplementation(Singleton)]` 实现全局注册表；消费方用
  `LifetimeScope.ResolveOptional<IWebPanelRegistry>()` **可选**注册，缺面板也照常工作。
- **动作种类**：`Table`（进页面自动加载）/`Form`（独立按钮命令）/`Search`（实时）/
  `Settings`（`Loader` 预填、页尾**统一保存**批量提交）/`Collection`（CRUD：`RecordsLoader`
  出 chip 列表 + 新增 + 点选编辑 + `DeleteHandler` 删除，按 `KeyField` upsert）。
- **Economy**：余额做成 `Collection`（按账户 set/delete；`ListAccountsAsync`/
  `DeleteAccountAsync` 加到 `ICurrencyBackend` 与两后端）；货币/后端、击杀奖励（经济来源）、
  转账做成 `Settings`，写回 economy `config.yaml`（`EconomyConfigStore`+`EconomyYaml`，内存权威副本）。
- **Shop**：商品目录做成 `Collection`（单条=物品、多条=礼包；chip 显示 `名称(ID)*数量`），
  游戏物品检索走 `IItemDirectory`，VIP 折扣做成 `Settings`；编辑重写 shop `config.yaml`
  （`ShopConfigStore`+`ShopYaml`，内存权威副本避免 reload 竞态）。全角符号解析归一化。
- **安全**：`bindAddress`/`token`；非回环且无 token 时自动降级回 127.0.0.1（除非
  `allowInsecurePublic`）。body 强制 UTF-8 解码。
- **打包/部署**：`scripts/build.sh` 新增平铺 `ProjectReference` 产物；各插件 `.csproj` 用
  `TargetsForTfmSpecificBuildOutput`+`BuildOutputInPackage` 把 `UnturnedMods.Shared.dll`
  打进自身 nupkg `lib/`。

## 为什么
- 让管理员在游戏外通过网页快捷管理余额/商品/经济设置；功能由各插件注册、不写死在面板里
  （延续「依赖抽象、不依赖具体插件」范式）。
