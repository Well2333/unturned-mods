# 新增面向玩家的实用插件 well404.Essentials（home/tp/warp/gift/sleep/back）

## 做了什么

- 新建插件 `src/plugins/well404.Essentials/`（脚手架生成后填实），面向**玩家**，
  把生存常用便利指令集中到一个插件，并共用一套传送流程。
- 命令：
  - `/home` 回家、`/home set` 设家（**单个家**，无命名/删除，按用户简化要求）。
  - `/tp <玩家>`：**同队直接传送**；**跨队**发请求，目标 `/tpa` 接受 / `/tpd` 拒绝，
    请求按 `tpa.expirationSeconds` 超时并通知发起者。
  - `/warp`、`/warp set <名> [冷却]`、`/warp delete`(别名 `del`)、`/warps`
    （指令格式参考 NewEssentials）。每个传送点用权限 `well404.essentials.warps.<名>`
    单独授权，可设独立冷却。`/warp delete` 允许控制台执行。
  - `/gift [id]`：免费礼包，按 **crontab 刷新规则**每周期可领一次，礼包可设
    可选 `permission` 做 VIP 专属。
  - `/sleep`：投票，在线玩家达 `sleep.requiredRatio`（默认半数）即切换昼/夜。
  - `/back`：回到死亡点 + 可配置秒数无敌。
- 统一传送流程 `TeleportService`：冷却检查 → （可选）经济费用预检 → 静止预热
  （移动取消，轮询位置）→ 传送 → **传送前原子扣费、失败退款** → 记冷却。
  费用仅在场上存在 `IEconomyProvider` 时生效，默认全 0（免费）；
  `well404.essentials.cooldown.exempt` 可免冷却。
- 可选集成（运行时解析，均无硬依赖）：经济 `IEconomyProvider`（收费）、
  `well404.WebPanel`（所有设置 + 传送点/礼包可视化编辑）。
- 持久化：玩家家/死亡点/礼包领取时间走插件 `IDataStore`（`players` 键）；
  传送点/礼包定义存 `config.yaml`（命令与 WebUI 共用 `EssentialsConfigStore`，
  与 `ShopConfigStore` 同构）。
- 单元测试 `tests/well404.Essentials.Tests/`：crontab 解析、礼包可领判定、时长
  格式化，共 33 条；全仓 `dotnet test` 绿（Economy 12 + Essentials 33）。
- 文档：填实插件 `README.md`、新增 `docs/well404.Essentials.md`、更新根 `README.md`
  与 `docs/README.md` 插件总表、并把 Essentials 行加入各插件 README 的「插件家族」表。

## 为什么

- 用户需求：制作一个围绕实用功能、面向玩家的多指令插件，并要求 warp 参考
  NewEssentials 指令格式、传送可选收费 + 预热静止、礼包支持 crontab 与 VIP 规则、
  sleep 投票切昼夜、back 带无敌。命名经确认为 `well404.Essentials`。

## 注意事项 / 后续

- **昼夜切换**（`DayNightController`）用 `LightingManager.time`（uint，`[0,cycle)`，
  `day=time/cycle`，`isDaytime = day < LevelLighting.bias`），按数据挖掘版语义实现
  并对退化 bias 做了保护；本地无游戏，**需在真实服务器实测**白天/黑夜切换是否准确。
- 标量设置（teleport/tpa/sleep/back）由各服务从 `IConfiguration` **实时**读取；
  传送点/礼包从 `EssentialsConfigStore` 读取（面板/命令写回 `config.yaml`）。手改
  `config.yaml` 的 warps/gifts 需 `openmod reload` 后生效（与 Shop 同样的取舍）。
- 冷却与待确认 tpa 请求为内存态，插件重载后清空。
- crontab 按**服务器本地时间**评估；领取时间以 Unix UTC 秒存储。
