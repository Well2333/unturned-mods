# 插件自建 WebUI、紧凑目录与商店/仓库交互升级

## 做了什么

- Shared 新增可选的插件自建管理端与玩家端 WebUI 扩展协议，同时保留原注册制；为 Shared 1.0
  已发布构造函数保留精确二进制兼容重载。
- WebPanel 以隔离的 Shadow DOM 挂载自建 UI，插件 API 限定在自身模块路径；管理端和玩家端
  均支持默认 5 秒定时刷新、Unturned favicon，以及仅在按下和释放均位于遮罩外时关闭弹窗。
- Essentials、Shop、Vault 迁移到自建 UI；其他插件继续使用兼容的注册制。
- Shop 新增分组标签、备注、拖动排序、紧凑六列商品目录、1/5/10/全部快捷购买与售卖、每组
  全部售卖和库存显示；未分组旧配置归入 `default`，无组名时以 ID 显示。
- Shop 移除组合包，默认目录改为瓶装水、豆子罐头、PDW、军用弹匣、军用小口径弹药箱、绷带、
  维生素和铁片，并可在服务端存在汉化资源时显示中文名。
- 修正 Shop 新商品价格保存、buy/sell 缺省数量、超量售卖，以及把弹匣内部弹药量误当商品数量
  的关键问题；库存与扣除现在按背包物品实例计数。
- Vault 同 ID 不同状态物品默认折叠为汇总卡，点击后在弹窗内显示各状态；仓库和 Shop 卡片使用
  六列紧凑布局并按窗口宽度降为 5/4/3/2/1 列。
- 文档补充双轨 UI 架构、Cloudflare 账号永久隧道、完整重启安装要求和新版操作说明。

## 为什么

- 复杂插件需要独立控制布局、交互和后续演进，简单插件仍可沿用低成本注册式界面。
- 原有宽卡片、单行记录和通用弹窗难以表达商店目录、仓库状态变体及批量操作。
- Shared 公共构造函数签名发生变化会导致未同步升级的插件在运行时出现
  `MissingMethodException`，因此本次明确补齐二进制兼容层。

## 守则同步

- 新增根目录必读 `Agent.md` 与 UI 基准 `DESIGN.md`，并把它们纳入长期记忆索引。
- 更新架构、开发、测试、工作协议和提交/发布守则：明确双轨 UI、发布必须逐次授权、用户未要求
  详细测试时以构建优先，以及远程专有信息只能保存在被 Git 忽略的本地记忆中。

## 注意事项

- 插件 DLL 更新必须完整重启服务器；`openmod reload` 只用于配置或生命周期验证，不替换二进制。
- Shared 必须只部署一份到 `OpenMod/plugins` 顶层，并先于本次其余四个包升级。
- TestServer 的专有地址、SSH 配置、管理 token、备份路径和操作记录仅保存在 `.local-memory/`，
  未进入公开提交。
