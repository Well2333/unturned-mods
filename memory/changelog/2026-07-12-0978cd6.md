# 修复 /sleep 错误跳到中午或午夜

## 做了什么
- 将昼夜切换目标从“目标时段中点”改为“下一个时段起点”：
  - 当前为夜晚：`LightingManager.time = 0`，跳到早晨/黎明；
  - 当前为白天：`LightingManager.time = LevelLighting.bias × cycle`，跳到傍晚/入夜。
- 抽出 `CalculateTargetTime` 纯函数，新增早晨、傍晚和异常 bias 回退三项回归测试。
- 同步 Essentials 包内 README 与详细文档，并将版本提升到 1.0.4。

## 为什么
- 旧实现取白昼区间中点与夜晚区间中点，因此夜晚投票后落在中午，白天投票后落在午夜；
  `/sleep` 的预期是跳过当前白天或黑夜，落到早晨或傍晚。

## 验证
- 默认 `cycle=3600`、`bias=0.6`：夜晚→`0`（早晨），白天→`2160`（傍晚），不再是
  `1080`（中午）或 `2880`（午夜）。
- `scripts/build.sh --test`：83 项单元测试全部通过，七插件 Release 构建成功。
- 本地 Test 服完整部署并整服启动：Essentials 1.0.4 与七插件全部正常加载，无插件异常；
  随后干净关服。
- 实际天空视觉和多人投票链路仍需在线玩家分别在白天、夜晚各执行一次 `/sleep` 验收。
