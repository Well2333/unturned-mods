# Economy 与 Vault 改用 SQLite，并补充永久 Cloudflare Tunnel 指南

## 做了什么

- 将 Economy 的 LiteDB 账本替换为规范化 SQLite 账户表和交易流水表，余额以 invariant
  decimal 文本保存，余额与审计记录在同一事务提交。
- 将 Vault 的运行数据从 `vault.data.yaml` 替换为 SQLite：
  `vault_items` 保存物品和原始状态 BLOB，`capacity_overrides` 保存玩家容量覆盖。
- 使用 `Microsoft.Data.Sqlite.Core` 与系统 `sqlite3` provider，避免 OpenMod 无法从
  NuGet 包解压 `e_sqlite3` 原生资产；短连接模式确保 reload 不长期锁库。
- Economy 数据库文件固定为 `economy.sqlite3`；经验后端不会创建、读取或写入 SQLite。
  Vault 数据库文件固定为 `vault.sqlite3`。
- 新增 Economy 和 Vault SQLite 测试，覆盖持久化、事务回滚、并发更新、精度、BLOB、
  容量覆盖、WAL 和原子取物。
- 更新手动部署构建脚本，平铺复制 `Microsoft.Data.Sqlite.dll`、
  `SQLitePCLRaw.core.dll` 和 `SQLitePCLRaw.provider.sqlite3.dll`。
- 版本更新：Economy 2.0.0、Vault 2.0.0、Shop 1.0.4、WebPanel 1.2.7。
- WebPanel 文档新增 Cloudflare 账号 Named Tunnel + 永久域名的 Linux/Windows 分步指南，
  包含 Host Header、安全说明和 400/502/无公网 URL 排障。

## 为什么

- LiteDB 与 YAML 全量重写不符合本仓库数据库插件统一使用 SQLite 的要求。
- SQLite 事务和规范化表能保证余额/流水、批量存物与取物等关键操作的一致性。
- OpenMod 的 NuGet 加载器不会展开 bundled `e_sqlite3` 原生库，因此改用系统
  `sqlite3` provider 才能在真实 Unturned/Mono 环境稳定加载。
- Quick Tunnel 域名会变化，长期服务器需要可复现的账号型固定域名配置。

## 注意事项 / 后续

- 本次明确不提供迁移或向后兼容：旧 `economy.db` 和 `vault.data.yaml` 不会被读取。
- 使用 Linux 系统 SQLite provider，服务器需提供 `libsqlite3`；已在真实本地
  Unturned/OpenMod/Mono 服务器验证加载、建表和 reload。
- 最终全仓 103 项测试通过；经验后端实测不会生成 `economy.sqlite3`。
- Shop 1.0.4 依赖 Economy 2.0.0，发布顺序必须先 Economy 后 Shop。
