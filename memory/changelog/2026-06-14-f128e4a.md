# WebPanel 通用化（去商店逻辑）+ 商店普通商品/礼包拆分（列表化玩家面 + 管理面一键加商品）

用户指出多项问题并要求按设计原则重审全仓库：①玩家面崩溃 `MissingMethodException`；②商店只保留
列表模式、删卡片；③管理面把普通商品与礼包拆开（普通商品只需 id+买卖价，礼包才需复杂设置），
商店插件同步；④**最重要**：WebPanel 不应包含任何其他插件的逻辑——任何插件功能都要自己注册才被
渲染，以保证任意插件组合都不出问题；⑤重读 guidelines 后对全部代码重新审核、重构修复。

## 做了什么

### ④WebPanel 彻底通用化（宿主零业务逻辑）
- **玩家面**：共享模型 `PlayerCard` 新增 `Group`（分区标题）+ `Badge`（前置短徽章）；
  `PlayerMenuView` 新增 `Layout`（`"list"`/默认卡片）；`PlayerButton` 样式增加 `success`。
  `player.html` 改为**纯通用渲染**（按 group 分区、list 紧凑行、badge、按钮样式），**删除**原先
  写死在宿主里的 `renderShop`/`buildShopTile`/`buildShopRow` 与 `menu.id==="shop"` 特判，以及
  shop 专属 STR；移除中途引入的 `Meta` 包。
- **管理面通用扩展**：`WebPanelAction.Hidden`（只可被 id 调用、不渲染卡片）+
  `WebActionResult.WithRowAction(actionId,label,rowKeys?)`（给 Table/Search 结果每行挂按钮，点击以
  该行 key 调用本模块某动作）。`index.html` 通用渲染之、跳过 hidden。
- ①崩溃根因：上一轮 `PlayerCard` 构造签名变动后只重部署了部分 DLL；本轮重建并全量重部署解决。

### ②③商店模型重构
- `config.yaml` 拆成 `items:`（普通商品：仅 itemId+buyPrice+sellPrice，显示名由游戏解析，
  `/buy /sell` 按**物品 ID** 引用，一次购买给 1 个）与 `bundles:`（id+name+buy+sell+contents）。
- 新增配置类 `ShopItemConfig`/`ShopBundleConfig` 与统一运行时类型 `ShopEntry`（由两份配置构建）；
  新增 `ShopNames` 集中「物品 ID→显示名」解析（命令/菜单/管理面共用）。
- `ShopCatalog.Find`：数字先按物品 ID 找普通商品，否则按 id 找礼包；`ShopConfigStore` 分别 CRUD
  items/bundles；`ShopYaml` 序列化两段；`ShopService.ItemsPerUnit` 普通商品每单位 1 个。
- 命令 `/buy /sell /shop` 改用 `ShopNames` 解析名称（注入 `IItemDirectory`）。
- **玩家商店改为列表化**：两分区（单品/礼包），经通用 `Group/Badge/Layout/Tags/Buttons` 驱动，
  买入按钮 `success` 绿色、价格拼进按钮 `Label`；**卡片模式删除**。
- **管理面拆成两个集合**：「普通商品」（itemId+买卖价）与「礼包」（id+名称+内容+买卖价）；
  「检索游戏物品」每行加「＋ 加入商店」（接到隐藏 `additem` 动作，建零价草稿商品）。

### ⑤全仓库审核（3 个子代理并行）
- Economy/Essentials/AdminTools：全部合规（§8 配置可 WebUI 编辑、全局服务硬规则、i18n、
  PlayerCard/MenuView 新签名调用、文档同步）——无需改动。
- WebPanel/Shared：宿主完全通用、无插件耦合；发现 1 处硬编码中文「不支持删除」绕过 i18n →
  已改走 `Tr("Delete is not supported here.")` 并补 ZhTable。
- Shop：代码正确、§8 完整、i18n 玩家面/管理面固定串均已挂键；发现 Shop README 配置示例仍是旧
  schema（§6 违规）→ 已更新为 items/bundles 两段。

## 为什么

- 把商店渲染写进 `player.html` 违反「WebPanel 不含具体插件逻辑」的根本设计；通用化后任意插件以
  注册数据驱动样式，组合不冲突。普通商品「只需 id+买卖价、名称自动解析」更省心；玩家面列表化最省
  空间、贴合 Steam 内嵌窄屏。
