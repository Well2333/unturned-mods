# 玩家/管理面板语言改存 YAML 配置文件（按玩家 id）

承接上一条(管理面语言改存服务端)。用户要求：玩家面板也做同样处理,且**存储方式放到配置文件里、
按玩家 id 存储**。

## 背景

玩家面其实早已是服务端按 Steam ID 持久化(旧 `player-languages.txt`,`steamId=lang` 行),且开屏采用
服务端返回语言、跨域名/新会话沿用——功能本就满足「同样的改动」。本次按用户要求只改**存储格式**:
从随手的 `.txt` 改成 **YAML 配置文件**。

## 做了什么

- `PlayerLanguageStore` → `player-languages.yaml`:
  ```yaml
  players:
    "76561198180246554": "zh"
  ```
  按玩家 Steam ID 存。新增简易 YAML 解析/写出(键值可带引号);**首次加载若无 .yaml 但有旧
  `player-languages.txt`,自动迁移**(读入后写出 .yaml),不丢已存偏好。
- `AdminLanguageStore` → `admin-language.yaml`(单值 `language: "zh"`),同样从旧 `admin-language.txt`
  自动迁移,保持两个面板存储格式一致。

## 为什么

- 快速隧道域名易变、浏览器存储按域名隔离会丢;服务端 YAML 配置文件按 id 存储更稳、更易查看。

## 影响的文件 / 范围

- `PlayerLanguageStore.cs`、`AdminLanguageStore.cs`(改 YAML + 迁移);`docs/well404.WebPanel.md`。

## 注意事项 / 后续

- **已验证**:测试服旧 `player-languages.txt`(`...554=zh`)与 `admin-language.txt`(`zh`)在加载时自动迁移为
  `player-languages.yaml`(`players: { "...554": "zh" }`)与 `admin-language.yaml`(`language: "zh"`);
  玩家/管理面 `?lang=en` 仍被已存 `zh` 压过;玩家切到 en→yaml 改写为 en,切回→zh。
