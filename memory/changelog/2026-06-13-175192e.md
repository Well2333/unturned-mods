# 玩家面五项改进：实用工具全展示 / 商店物品名 / 首页置顶 / 语言记忆 / 纯数字物品检索

用户在游戏内继续测试时提出的五点。

## 做了什么

- **①实用工具(Essentials)所有功能常驻可见**。原菜单完全按上下文渲染：玩家独自在空服上只看到
  「家 + 睡觉投票」两张卡,以为功能没加。现在每个工具在无数据时也渲染一张带提示的空态卡：
  返回死亡点、传送点、队伍(含「你还没有队伍」提示)、其他在线玩家、礼包——全部常驻。
- **②玩家商店显示物品名而非数字 ID**。原先标签是 `#15×1` 这种原始 ID,且「礼包」徽章与内容标签
  同行同样式。现在经 `IItemDirectory` 把 ID 解析为物品名：单物品卡片不再显示冗余标签(标题即物品名);
  组合包单列「礼包 —— 包含：」一行,下面用 **物品名 ×数量** 标签逐项列出,与普通物品视觉区分。
- **③「首页」标签置顶并作为开屏默认页**。`PlayerMenuRegistry.GetMenus` 现在把 home/intro 标签排在
  第一(其余保持注册顺序,OrderBy 稳定),开屏即落在首页而非钱包;`/menu shop` 仍可定位到指定标签。
- **④玩家语言选择服务端记忆**。新增 `PlayerLanguageStore`,按 Steam ID 持久化到
  `player-languages.txt`。玩家 API 用「已保存语言」覆盖查询串;新增 `POST /api/p/lang` 保存切换;
  `player.html` 切换时先保存再刷新——下次任意设备打开都沿用该语言。
- **⑤管理面「检索游戏物品」纯数字精确匹配**。Shop 与 Essentials 的物品检索,在输入为纯数字时
  优先把**精确 ID** 的物品列在最前,即使大量 ID 含这些数字也能找到。

## 为什么

- 均为用户上服测试反馈;尤其①是「为什么实用工具就两个」的真实原因(上下文折叠),非功能缺失。

## 影响的文件 / 范围

- ①:`EssentialsPlayerMenu.cs`(各段加空态卡)、`Essentials/WebI18n.cs`(空态文案)。
- ②:`ShopPlayerMenu.cs`(注入 `IItemDirectory` + `BuildNameMapAsync`/`ItemLabel`,改标签/行)、
  `ShopPlugin.cs`(传 itemDirectory)、`Shop/WebI18n.cs`。
- ③:`PlayerMenuRegistry.cs`(home 置顶)。
- ④:`PlayerLanguageStore.cs`(新)、`WebPanelHttpServer.cs`(saved-lang + `/api/p/lang`)、
  `WebPanelPlugin.cs`(构造 store)、`player.html`(切换即存 + 采用服务端语言)。
- ⑤:`ShopWebPanelModule.cs`、`EssentialsWebPanelModule.cs`(纯数字精确 ID 置顶;补 `using System.Linq`)。
- 文档:`docs/well404.WebPanel.md`、`docs/well404.Shop.md`、`docs/well404.Essentials.md`。

## 注意事项 / 后续

- 服务器实测(端口 27020)：5 插件全部加载、无异常;**⑤已 curl 验证**——Shop 检索 `15` 返回
  `Medkit(15)` 在首位,随后才是 1511/1533… 等子串匹配。
- ①②③④属**玩家面**(需玩家在游戏内 `/menu` 生成会话才能打开),无法用 curl 验证;代码已编译通过、
  插件注册正常。需用户在游戏内复验:实用工具是否各功能都显示、商店是否显示物品名、开屏是否落在
  首页、切换语言后重开是否沿用。
- 经验:玩家菜单若「按上下文」渲染,空服会塌成很少几张卡,易被误解为功能缺失——**关键功能应常驻
  并给空态提示**。玩家展示一律用物品名而非数字 ID(经 `IItemDirectory`)。玩家偏好(如语言)要持久化
  在 WebPanel 工作目录,跨会话/设备生效。
