# 新增 well404.AutoSave：cron 定时保存 + LZMA 高压缩存档备份（squash 合并自 feat/autosave）

## 做了什么
- 新增插件 `well404.AutoSave`（1.0.0），按 **cron 墙钟**触发 `SaveManager.save()`
  （默认 `*/10 * * * *`，不随启动计时），每保存 N 次（默认 6）做一次备份。
- 备份为 **tar + LZMA 实体压缩**的 `.tar.lz`（SharpCompress `TarWriter` over `LZipStream`），
  先写 `*.tmp` 再原子改名；信号量防并发。
- **备份范围**：默认整个 `Servers/<id>/` 减去可下载/可再生内容
  （`Workshop/**`、`Steam/**`、`Bundles/**`、`OpenMod/packages/**`、`**/logs/**`、`*~`、`*.bak`）；
  排除规则为相对存档根的 glob（`*` 不跨目录、`**` 跨目录、大小写不敏感），可配。
  备份目录可自定义（默认 `<安装根>/Backups/<serverId>`，在存档目录外、并自排除避免套娃）。
- **保留策略**：最大数量 + 最大总体积，任一超限删最旧，默认不限且永远保留最新一份。
- **无游戏内指令**（按用户要求）：配置走 `config.yaml`；装 `well404.WebPanel` 时提供
  设置组（全部配置 WebUI 可编）+ 备份列表（行内删除）+ 立即备份，中英双语。
- 保存计数持久化（`save-count.txt`），跨重启/`reload` 延续备份节奏。
- 新增依赖（集中在 `Directory.Packages.props`）：`Cronos` 0.8.4、`SharpCompress` 0.30.1
  （netstandard2.1 最小依赖闭包，仅额外 `System.Text.Encoding.CodePages`，已显式引用以便
  `build.sh` 扁平拷贝）。
- 单测 34 项（cron 取下次时间、exclude glob、保留策略、`.tar.lz` 打包/解包往返）。

## 为什么
- 用户需求：定时保存与备份，cron 触发，高压缩省体积，备份目录可自定义、可设最大数量/体积，
  默认不限；只保留必要存档、排除创意工坊等可下载内容；不要游戏内指令、仅 WebPanel + 配置。
