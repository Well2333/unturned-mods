# WebPanel:修复默认 127.0.0.1 绑定下经隧道访问全部返回 400 (Invalid host)

## 做了什么(well404.WebPanel 1.2.1 → 1.2.2)
- cloudflare 预设参数加上 `--http-host-header 127.0.0.1:{port}`,让 cloudflared 转发到本地面板时发送
  `Host: 127.0.0.1:{port}`,匹配默认 `bindAddress: 127.0.0.1` 的 `HttpListener` 监听前缀。
- 仅影响内置 cloudflare 预设;`type: custom` 不变。

## 为什么
- 真机部署(默认 `127.0.0.1` 绑定)经隧道访问时,cloudflared 默认转发的是**公网 trycloudflare Host**,
  而 `HttpListener` 绑定 `http://127.0.0.1:<port>/` 只接受 `Host: 127.0.0.1:<port>`,Host 不匹配即回
  `400 (Invalid host)` —— 导致经隧道的**每个**请求(含根 `/` 与玩家 `/p`)全部 400。
- 用 `--http-host-header` 让代理发送匹配的 Host,**保留回环绑定的安全性**,无需把 `bindAddress` 改成
  `0.0.0.0` 或开放入站端口。cloudflared 官方该参数即为「设置本地源站的 Host 头(配合 --url)」。
- 此前隧道因 cloudflared 下不下来从未真正跑通,故该潜在 bug 未暴露;之前本地测试又恰用了
  `bindAddress: 0.0.0.0`(通配前缀 `+`,任意 Host 都收),也未踩到。真机用默认绑定跑通隧道后才暴露。

## 影响的文件 / 范围
- `well404.WebPanel/WebPanelPlugin.cs`(`ResolveEffectiveTunnel` 的 cloudflare 预设 Args)
- `well404.WebPanel/docs/well404.WebPanel.md`(说明)、`.csproj`(1.2.2)
