# 统一物品元数据、双语显示与 Vault 目录筛选

## 做了什么

- Shared 新增统一的 `LocalizedItemCatalog`，从原版与创意工坊资源解析中英文名称、
  `ItemAsset.showQuality`、原生物品类型、稀有度、稀有度等级和玩家筛选类别。
- WebPanel 为 `PlayerCard` 增加不透明字符串 `Metadata`，保留旧构造函数的精确二进制兼容；
  通用管理端与玩家端按面板语言显示中文主标题和灰色英文副标题。
- Essentials 与 Shop 改用共享物品目录，中英文名称均可检索；中文界面显示中英双语，
  英文界面只显示英文。
- Vault 只在资源明确启用 `showQuality` 时显示并匹配耐久，修复无耐久物品被原始 quality
  字节拆分和显示虚假百分比的问题，同时保持数据库原始数据完整保真。
- Vault 物品卡按原生稀有度显示彩色边框和文字；新增总占用格数、ID、数量、稀有度、
  名称排序与升降序切换，默认按总格数降序。
- Vault 新增弹药、食物与饮品、药品、武器、材料、工具、服装、配件、建筑、载具用品和
  其他分类筛选；Supply 弹药箱通过 AMMO 蓝图语义归入弹药，不依赖名称或 ID 猜测。

## 为什么

- Unturned 每个物品实例都有原始 quality 字节，但只有 `showQuality` 为真时它才代表玩家可见的
  耐久；按字节无条件展示会让夹板、医疗品和弹药箱产生虚假耐久。
- 物品翻译、类别和稀有度属于跨插件公共元数据，集中解析可以避免 Shop、Vault、Essentials
  各自实现后产生不一致。
- 大仓库需要稳定排序和原生类别过滤，才能在紧凑六列布局中快速定位物品。
