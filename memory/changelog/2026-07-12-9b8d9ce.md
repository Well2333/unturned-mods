# 修复 NuGet 插件注册、玩家操作与管理工具

## 做了什么

- 将 `UnturnedMods.Shared` 改为唯一的 `well404.UnturnedMods.Shared 1.0.0`
  NuGet 包，所有插件改为依赖该包，不再各自内嵌 Shared.dll。
- 修复 WebUI 同队传送，使其与游戏内 `/tp` 一致：同队直接传送，跨队才发送确认请求。
- 为 AdminTools 新增 `/repair <player>` 与网页面板“维修装备”，把指定在线玩家的武器、
  衣物和可维修配件恢复到 100%，并保证枪械弹药字节不被误改。
- 修复 Shop 的 `/buy <id> [amount]` 和 `/sell <id> [amount]` 参数解析：
  省略数量时明确默认为 1，非法数量仍会被拒绝。
- 修正 Essentials 与 AdminTools 文档中的 OpenMod 命令权限节点为 `commands.*`。
- 增加 AdminTools 和 Shop 回归测试，并补齐 Shared 的版本、README、发布工作流和发布脚本支持。

## 为什么

- OpenMod 会逐个加载每个 NuGet 包内的程序集；旧包各自携带 Shared.dll，产生多个程序集身份，
  导致 WebPanel registry 与业务插件接口不相同，表现为面板只剩首页、reload 后功能消失。
- WebUI 传送没有复用游戏内指令的同队直传分支，Shop 数量缺省也不应依赖框架的泛型参数绑定。

## 影响的文件 / 范围

- Shared 与七个插件的打包依赖和版本。
- Essentials 玩家网页菜单与权限文档。
- AdminTools 命令、服务、网页管理模块和测试。
- Shop buy/sell 命令、文档和测试。
- GitHub Actions、版本/Release 脚本、插件模板和解决方案。

## 注意事项 / 后续

- 发布时必须先发布 Shared 1.0.0，再发布依赖它的插件版本。
- 已在隔离测试服通过真实 OpenMod NuGet 安装和 `openmod reload` 验证：
  reload 前后均注册首页、钱包、实用工具、商店、仓库，Cloudflare Tunnel 也重新取得公网 URL。
- 同队 TP 与实际在线玩家物品维修仍需在生产服由在线玩家完成最终游戏内验收。
