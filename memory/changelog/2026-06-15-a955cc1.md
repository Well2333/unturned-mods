# 仓库菜单重做：相同合并 / 不同属性可折叠就地展开 / 全部·数量·一件

用户对上一版的反馈:①存入把同类一次全存了、背包没法看详情挑哪个;②应支持「全部取出/取出X个/取出一个」
(存入同理);③二级菜单不该把其它都隐藏,要像**折叠卡片**点击就地展开,信息量更大;④**无属性或属性完全
一致的物品应合并**、不设展开;不同属性的同种物品展开时把**相同部分合并**。

## 做了什么

- **按变体合并**:把「id+数量(弹药)+耐久+状态字节」完全相同的若干件视为同一变体合并成一行(显示 `×件数`)。
  某物品只有一个变体 → 单张卡片、无展开;有多个变体 → 折叠父卡片 + 子卡片(每个变体一行,仅显示其**区别属性**:
  `数量 N`/`耐久 X%`/`🔧`)。
- **就地折叠(不隐藏其它)**:新增通用 `PlayerCard.Children`,客户端渲染「展开/收起」并把子卡片缩进显示在父行下,
  其余条目仍在;展开状态用 JS `Set` 跨重渲染保留(取完一件后仍保持展开)。替换掉上一版「整页替换」的下钻。
- **三种操作**:每行「全部 / 数量 / 一件」(存与取各一套,按区分颜色 primary/success);变体行只作用于**该变体**
  (于是 5 发盒与 8 发盒可分别批量存取)。**背包同样处理**,可挑选要存哪个变体。
- 服务层:`ItemVariant` + `VaultVariants`/`BackpackVariantsAsync`(按变体分组)+ `StoreVariantAsync`/
  `TakeVariantAsync`(谓词过滤匹配变体);`StoreMatchingAsync`/`TakeMatchingAsync` 复用。移除 uid 下钻
  (`TakeByUidAsync`)与旧 `ListBackpackAsync`/`BackpackEntry`。变体键编码 `id|amount|quality|base64state`。

## 影响的文件 / 范围

- Shared:`PlayerMenu.cs`(`PlayerCard.Children`)。
- WebPanel:`WebPanelHttpServer.cs`(递归序列化 children,抽出 `AppendPlayerCards`)、`player.html`
  (`expandedRows` Set + buildRow 折叠渲染 + `.prowwrap`/`.children` 样式 + `details` 文案)。
- Vault:`VaultService.cs`(变体模型与方法)、`VaultPlayerMenu.cs`(整改:合并/折叠/三操作/变体键)、
  `VaultI18n.cs`(全部/数量/一件/数量 {0})、`docs/well404.Vault.md`、`README.md`。
