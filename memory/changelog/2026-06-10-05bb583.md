# 初始化 OpenMod Unturned 多插件 monorepo 与长期记忆系统

## 做了什么
- 搭建多插件仓库骨架与统一构建环境：
  - `UnturnedMods.sln`、`Directory.Build.props`（统一 `netstandard2.1`、Nullable、
    警告策略、包元数据）、`Directory.Packages.props`（集中式依赖版本管理 CPM）、
    `global.json`、`nuget.config`、`.gitignore`。
  - 跨插件共享库 `src/Shared/UnturnedMods.Shared`。
  - 插件模板 `build/templates/plugin/` + 脚手架 `scripts/new-plugin.sh`。
- 落地仓库内**两级长期记忆系统**：
  - 一级 `memory/guidelines/`（规范、架构、提交/记忆流程）。
  - 二级 `memory/changelog/`（本文件即首条记录）。
- 新增 `README.md`、`CLAUDE.md`（AI 协作入口，要求开工前先读一级规范）。
- 依据 NuGet 现状将 `OpenMod.Unturned` 锁定为 `3.8.10`、`Legacy2CPSWorkaround`
  为 `1.0.0`。

## 为什么
- 用户要求基于 OpenMod 开发若干 Unturned 插件，需可容纳多插件、运行环境一致；
  并要求把长期记忆存于仓库内、分两级、规范变更及时同步、常规变更逐次记录
  （文件名 `YYYY-MM-DD-<commithash>.md`）。

## 影响的文件 / 范围
- 全仓库初始化（详见首次提交 `05bb583` 的文件列表）。

## 注意事项 / 后续
- **当前环境仅安装 .NET 运行时 8.0.27，未安装 .NET SDK**；构建插件前需先装 SDK
  （建议 8.0 LTS）。装好后可用 `dotnet build` 验证脚手架生成的插件可编译。
- 工作目录原属 root，已 `chown` 给 `well404` 以便正常读写。
- `OpenMod.Unturned 3.8.10` 为撰写时 NuGet 最新稳定版（发布于 2024-07-10）；
  后续应核对是否有更新版本再升级。
- 本仓库内记忆为权威来源，独立于 AI 运行环境的私有记忆区。
