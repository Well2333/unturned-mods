# 仓库:弹药装填比例(6/8)+「数量」按钮区别于「全部」+ 物品双列网格

承接上一轮(a955cc1)仓库重做后的三点反馈:

1. **弹药/堆叠显示成「数量 6」易与件数(×N)混淆** → 改成**装填比例**(如 `6/8`,当前/容量)。
2. **「数量」按钮点击效果和「全部」一样** → 修复。
3. **物品布局浪费空间(带标签的行被强制占满整行 → 实际单列)** → 改成**响应式双列网格**。

## 做了什么

- **装填比例**:`StoredItem` / `ItemVariant` 新增 `MaxAmount`(弹匣/弹药盒容量,取自
  `ItemMagazineAsset.amount`)。存入时(主线程)捕获;`InitializeAsync` 对历史数据(`Amount>1` 且
  `MaxAmount==0`)回填一次,使 Web 线程渲染时可直接读到容量。变体标签:`MaxAmount>1` 时显示
  `Amount/MaxAmount`(如 `6/8`),容量未知则退回旧的 `数量 N`。
- **「数量」区别于「全部」**:
  - 某变体/物品**只有 1 件**时,只渲染**一个** `存入/取出` 按钮(此时 全部=数量=一件 全等,冗余)。
  - **多件**时仍是 `全部 / 数量 / 一件`,但「数量」弹窗默认值由「总件数」改为 **`1`**——需玩家自行填写,
    因此快速点击不再等同「全部」。
- **双列网格**:通用玩家菜单 `list` 布局不再让带标签的行 `grid-column:1/-1` 占满整行;改为
  `repeat(auto-fill,minmax(320px,1fr))` 的响应式网格,单元内把「徽章+名称+标签」(`.ptop`)叠在操作
  按钮之上。可折叠父卡片仍占满整行,但其**子变体**(`.children`)也用 `minmax(280px,1fr)` 双列网格排布。

## 影响的文件 / 范围

- Vault:`VaultData.cs`(`StoredItem.MaxAmount`)、`VaultService.cs`(`ItemVariant.MaxAmount`、
  `MaxAmountOf`、存入捕获 + 初始化回填)、`VaultPlayerMenu.cs`(比例标签、单件单按钮、数量默认 1)、
  `docs/well404.Vault.md`、`README.md`。
- WebPanel:`player.html`(`buildRow` 改为 `.ptop` 叠按钮的单元 + 移除 `.prow.wide`;`.plist` /
  `.children` 双列网格)。这是**通用渲染器**,商店玩家菜单同样受益(已截图确认商店变为双列且无破版)。

## 验证 / 注意

- 用用户**真实仓库数据**(只读 dev 预览 + Chrome CDP 截图)验证:两盒 12 Gauge Shells 展开为 `8/8`、`5/8`
  两个变体(各一个 `取出` 按钮),并排双列;Medkit ×3 合并为一张卡,带 `全部/数量/一件`(数量默认 1)。
  商店页确认 单品 5 项 / 礼包 1 项 均双列排布、买卖按钮在下方。
- **`数量` 默认 1** 是当前取舍:对「多件」条目,玩家点「数量」需自行输入个数;若想一次取全请用「全部」。
- **仍待在线实测**:实际存/取某变体走的是已实测的搬运机制 + 变体谓词过滤;部署用 `openmod reload`
  完成,用户当前离线,变体存/取的端到端仍需其重连后再验。
- 仅改 Vault.dll + WebPanel.dll(Shared 未动,无 MissingMethodException 风险);`player.html` 为
  WebPanel 内嵌资源,故 WebPanel.dll 需随之重新部署。
