# 管理面板语言改为服务端持久化（不再仅存浏览器）

用户(附 DevTools 截图：localStorage `adminpanel_lang=zh`)指出：管理面语言只存在浏览器 localStorage,
按域名隔离;由于管理面 URL 走快速隧道、域名每次重启随机变,换链接后 localStorage 丢失,语言就重置回英文。
要求把该设置存到服务器内。

## 做了什么

- 新增 `AdminLanguageStore`：把管理面语言以单值存于工作目录 `admin-language.txt`(管理面只有一个 token、
  无按用户身份,故单值即可),线程安全。
- 服务端：`GET /<token>/api/modules` 现用「**已保存的管理面语言** ?? 查询串」来本地化并回填 `lang`
  字段(已保存语言优先于 `?lang=`);新增 `POST /<token>/api/lang` 保存。
- 前端 `index.html`：加载时**采用服务端返回的 `data.lang`**(不再仅在 localStorage 为空时);切换语言时
  先 `POST /api/lang` 持久化再刷新;localStorage 降级为缓存。

与玩家面早已有的「按 Steam ID 存 `player-languages.txt`」对称——现在两个面板的语言都存服务端。

## 为什么

- 快速隧道域名易变,浏览器存储按域名隔离,导致设置丢失;服务端持久化才稳。

## 影响的文件 / 范围

- 新 `AdminLanguageStore.cs`;`WebPanelHttpServer.cs`(构造参数 + `/api/lang` 路由 + modules 用已存语言)、
  `WebPanelPlugin.cs`(构造 store)、`index.html`(采用服务端语言 + 切换即存)。
- 文档：`docs/well404.WebPanel.md`。

## 注意事项 / 后续

- **已验证**:`POST /api/lang lang=zh` → `admin-language.txt` 写入 `zh`;`/api/modules?lang=en` 仍返回
  `lang=zh`、模块标题为中文(已存语言压过查询串);**全新 Chrome 配置(空 localStorage,等同新隧道域名)
  打开管理面直接是中文**(截图确认)。
