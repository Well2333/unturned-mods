# 守则：推送/发版需用户当次明确授权

## 做了什么
- 在 `commit-and-memory-workflow.md` 增加「最高红线」：`git push`、创建 tag/Release、
  发布 NuGet（`release-plugin.sh`、publish workflow）只能在用户**当次**明确要求时执行，
  不得擅自；一次授权不延续到后续改动。本地 `git commit` + 变更记录可进行，但停在本地。
- 在 `publishing.md` 顶部加醒目提示，指向该红线。

## 为什么
- 用户要求：未明确指示时不要擅自推送 GitHub / 发版（控制对外、不可逆操作）。

## 影响的文件 / 范围
- `memory/guidelines/commit-and-memory-workflow.md`、`memory/guidelines/publishing.md`。

## 注意事项 / 后续
- 本次按用户明确指令提交并推送（不发版）。

## 规范同步（guideline_changed = true）
- 同步更新上述两份一级规范，新增「推送/发版需当次授权」红线。
