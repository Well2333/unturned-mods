# 发布认证改用 NuGet Trusted Publishing（OIDC，免长期密钥）

## 做了什么
- `.github/workflows/publish.yml`：
  - job `permissions` 增加 `id-token: write`（允许签发 GitHub OIDC 令牌）。
  - 移除对 `secrets.NUGET_API_KEY` 的依赖与缺失校验，改为：先 `NuGet/login@v1`
    （`user: Well404`，nuget.org 用户名）用 OIDC 换取临时 API Key，紧接着
    `dotnet nuget push --api-key <临时key> --skip-duplicate`。
- 同步更新一级规范 `memory/guidelines/publishing.md`（见下）。

## 为什么
- 免去维护长期 `NUGET_API_KEY` 密钥，降低泄露风险；临时 key 有效期 1 小时、一次性。
- 本仓库为 public，Trusted Publishing 策略首发即永久生效；账号也已可见该功能。
- 正好绕开"NUGET_API_KEY 未配置"这一发布前阻塞。

## 影响的文件 / 范围
- `.github/workflows/publish.yml`
- `memory/guidelines/publishing.md`

## 注意事项 / 后续
- **一次性前置（网页操作，需账号所有者）**：nuget.org → Trusted Publishing → 新建策略：
  Owner=`Well2333`、Repository=`unturned-mods`、Workflow File=`publish.yml`（只填文件名）、
  Environment 留空。策略按账号所有者维度生效，覆盖 `well404.*` 全新包 ID 的首发。
- `NuGet/login` 的 `user:` 写死为 `Well404`（非敏感、公开）；改用户名/owner/workflow
  文件名时须同步改 nuget.org 策略与该字段。
- 回退：若某账号未开放该特性，改回 `gh secret set NUGET_API_KEY ...` 并把 push 步骤
  恢复为使用 `secrets.NUGET_API_KEY`。
