# 添加玩家仓库管理与 Essentials 分类导航

## 做了什么

- Vault 2.4.0 在管理员面板新增按 SteamID 检索玩家仓库、精确编辑/删除记录及按物品 ID 批量删除；所有删除均有二次确认，服务端按玩家和记录双重限定。
- Vault 的 SQLite 写路径保留物品状态数据，并重新计算格子占用与最大堆叠信息；新增跨玩家隔离、精确删除和批量删除测试。
- Essentials 1.3.0 将在线玩家、请求和队伍合并为“玩家与队伍”，并为邀请、接受、退出和踢人操作增加确认。
- 传送点新增标签与稳定顺序：管理员可按标签查看并拖动排序，玩家只能查看标签及过滤。
- 旧传送点配置会在插件加载后自动写入 `category: default` 和顺序，`/warp set` 保留已有标签及顺序。
- 同步更新两插件 README、独立文档、前端静态检查与回归测试。

## 为什么

- 让管理员能够在不直接操作生产 SQLite 文件的情况下安全处理指定玩家的仓库记录。
- 降低玩家与队伍功能的导航成本，并让大量传送点可按用途分类和稳定排序。
- 保持旧配置无须人工迁移即可使用新版面板。

## 影响的文件 / 范围

- `src/plugins/well404.Vault/**`、`tests/well404.Vault.Tests/**`、`docs/well404.Vault.md`。
- `src/plugins/well404.Essentials/**`、`tests/well404.Essentials.Tests/**`、`docs/well404.Essentials.md`。

- 全仓 Release 测试 171/171 通过；Vault 38 项、Essentials 42 项均通过，两个自建 UI 脚本语法检查通过。
- 本地 `.localserver` 已验证 Essentials 1.3.0 加载、7 插件共同加载及旧 warp 配置写回；测试后立即关服并恢复原配置。
- 管理员实际删除玩家物品仍应先核对 SteamID 和物品 ID，并使用界面二次确认；本次未直接修改任何生产仓库数据。
