# 仓库:「数量」弹窗改为空白输入,彻底区别于「全部 / 一件」

承接 ce141e9 后用户反馈:「数量」按钮**又变成「一件」了**(并表示修不好就把它去掉)。

## 根因

「数量」是一个带默认值的 `window.prompt`:默认值若取「总件数」→ 快速确定 = 「全部」;上一版改成默认 `1`
→ 快速确定 = 「一件」。**任何固定默认值都会和「全部」或「一件」重合**。

## 做了什么(选择修复而非删除,以保留早前明确要求的「取出 X 个」)

- 「数量」按钮的 `promptDefault` 改为**空字符串 `""`** → 弹窗**留空**,玩家必须自行输入数字,因此**与「全部」
  「一件」在构造上都不同**。提示标题追加可填范围,如 `取出数量 (1–3)`。
- 通用客户端(`player.html` 的 `invoke`)改为**按需空白**:只有显式 `promptDefault === ""` 才打开空白框;
  默认缺省(`null`/`undefined`)仍回退 `"1"`。因此**商店的「出售数量」弹窗不受影响**(它没传默认值)。
- 单件条目仍只有一个「存入/取出」按钮(沿用 ce141e9)。

## 影响的文件 / 范围

- WebPanel:`player.html`(`invoke` 默认值解析:`(promptDefault==null)?"1":promptDefault`)。
- Vault:`VaultPlayerMenu.cs`(`OpButton("some")` → `promptDefault=""` + 标题加 `(1–N)`)、
  `docs/well404.Vault.md`、`README.md`。

## 验证 / 注意

- **CDP 拦截弹窗确认**:点「数量」弹出 `type=prompt`、`message='取出数量 (1–3)'`、`defaultPrompt=''`(空)。
- 仅改 Vault.dll + WebPanel.dll(Shared 未动);`player.html` 为 WebPanel 内嵌资源,故 WebPanel.dll 同步重新部署。
- 商店「出售数量」未传默认值 → 仍回退 `1`,行为不变(刻意保持)。
- 实际「取出 X 个」的搬运是在线操作(dev 预览离线只读),走的是已实测的搬运 + 变体过滤;用户重连后可再验。
