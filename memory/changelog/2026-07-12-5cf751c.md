# 修复动态传送点权限注册与 reload 后 Cloudflare Tunnel 恢复

## 做了什么
- Essentials 在启动时注册所有已有传送点的动态权限，并在游戏命令或 WebPanel 新增、编辑
  传送点时立即注册对应权限。
- 传送点权限检查和文档统一使用完整 OpenMod 节点
  `well404.Essentials:well404.essentials.warps.<name>`；同时注册固定的传送冷却豁免权限。
- 增加 Unicode 传送点权限回归测试，覆盖“绿洲”节点的注册和完整节点检查。
- WebPanel 将动态 tunnel URL 状态按插件加载代次隔离，reload 时清除旧 URL，并阻止旧实例的
  迟到回调覆盖新实例状态。
- Cloudflare Quick Tunnel 首次启动未取得 URL 时，在 `autoRestart: true` 下每 3 秒重试；
  `/menu` 等待时间与 `readyTimeoutSeconds` 对齐，并补强 unload/URL 同时发生的进程竞态。
- Essentials 版本提升至 1.0.2，WebPanel 版本提升至 1.2.5。

## 为什么
- OpenMod 的权限检查器会拒绝任何未先注册的权限，原有 per-warp 动态节点只生成、未注册，
  导致玩家菜单、`/warp` 和 `/warps` 抛出 `Permission is not registered`。
- `openmod reload` 后 cloudflared 首次进程可能快速退出；旧实现会永久放弃本次 tunnel，且全局
  玩家链接服务跨 reload 存活，存在旧状态或旧回调污染新实例的问题。

## 影响的文件 / 范围
- `src/plugins/well404.Essentials/`：权限注册、WebPanel warp 写入路径、配置注释、README、版本。
- `src/plugins/well404.WebPanel/`：tunnel 生命周期、重试、玩家链接状态、README、版本。
- `docs/well404.Essentials.md`、`docs/well404.WebPanel.md`：管理员权限与 reload 行为说明。
- `tests/well404.Essentials.Tests/WarpPermissionTests.cs`：动态权限回归覆盖。
