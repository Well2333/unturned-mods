# WebPanel 隧道自动保活：掉线后自动重启并启用新地址

用户反馈:服务器开一段时间后,WebPanel 的 Cloudflare 链接会失效,而服务器并不会刷新它,网页
「connection closed」。要求修复。

## 做了什么

- 新增**隧道监控**(后台 Task,`web.tunnel.autoRestart` 默认 true):周期性检查隧道,以下任一即
  **重启 cloudflared 并启用新地址**——
  - 隧道**进程退出**(立即发现,经 `ITunnelProvider.IsRunning`);
  - 隧道**公网 URL 不再响应**(对 `<url>/` 发 HTTP 探测,非 2xx/超时算不健康)。
  重启后调 `sessions.SetTunnelBaseUrl(newUrl)` 更新玩家链接基址,并打印新管理面地址(快速隧道地址
  每次会变,玩家需重新 `/menu`)。
- **探测自校准防回环**:HTTP 探测必须**至少成功一次**(probeProven)才被允许触发重启;若本机根本连不通
  该 URL(如无出站 HTTPS),探测自动停用,只靠进程退出检测,**绝不**陷入反复重启。失败需连续 2 次才重启。
- `web.tunnel.healthCheckSeconds`(默认 60,0=只在进程退出时重启)控制探测/检查频率。
- `ITunnelProvider` 加 `IsRunning`;`ProcessTunnelProvider` 实现;`StartTunnelAsync` 接受
  CancellationToken;卸载时取消监控 CTS 并停隧道。`ResolveEffectiveTunnel` 透传两个新字段。

## 为什么

- 快速隧道是临时的,跑几小时可能被 Cloudflare 回收/掉线;原代码只在启动时拉一次、之后不管,导致
  长期不可用且需手动重启服务器。

## 影响的文件 / 范围

- `WebPanelSettings.cs`(`AutoRestart`/`HealthCheckSeconds`)、`TunnelProvider.cs`(`IsRunning`)、
  `WebPanelPlugin.cs`(`MonitorTunnelAsync`/`ProbeAsync` + 启动监控 + 卸载取消 + 透传)、
  `config.yaml`(注释两个新项)、`docs/well404.WebPanel.md`。

## 注意事项 / 后续

- **已实测**(测试服把 `healthCheckSeconds` 临时设 15):`kill -9` cloudflared 后约 15s 内日志出现
  「tunnel process is no longer running — restarting it」→「tunnel up at <新URL>」→「tunnel restarted
  — new admin panel URL …」,新 cloudflared 进程拉起;**全新启动**的隧道根路径 `curl` 返回 200;
  **无重启回环**(把刚重启、短暂 530 的新 URL 不会触发再次重启——自校准生效)。
- 现象说明:**极快地** kill+重启 会让新隧道短时间 530(Cloudflare 重注册节流),但全新启动是 200;
  真实场景(隧道跑几小时后掉线再重启)等同全新启动,可正常服务。
- 这两个是隧道基础设施配置,按 development-standards §8 例外**不进 WebUI**,只改 config.yaml。
