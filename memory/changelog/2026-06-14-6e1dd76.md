# 玩家商店电商化重设计：单品/礼包分区 + 卡片/列表双视图

用户反馈(附截图)：上一版商店「空间浪费依旧严重」（卡片过大、跨类卡片等高拉伸留下大片空白），
要求参考电商页面逻辑优化，**新增列表模式**，并把**礼包与独立物品分开放置**。考虑到是 Steam 内嵌
浏览器（窄屏）。

## 做了什么

- **单品 / 礼包分区**：玩家商店(`/p` shop)把 `kind=item` 与 `kind=bundle` 拆成两个独立分区
  （各带数量计数），不再混在一个 `align-items:stretch` 的网格里被拉伸出空白。
- **两种视图**（右上角「卡片/列表」切换，记忆在 `localStorage.shop_mode`）：
  - **卡片**：电商式商品瓦片网格（`minmax(168px)`，窄屏自动减列）；物品名 + 大号绿色买价（有折扣
    显示折后价 + 划线原价）+ 购买/出售按钮；礼包瓦片用胶囊标签逐项列内容。
  - **列表**：紧凑行，响应式单/双列（`minmax(310px)`）。单品行 = `#ID | 物品名 | 购买 $买价 |
    出售 $卖价`（价格即点击买/卖的按钮）；礼包行 `grid-column:1/-1` 整行更宽 = `礼包名 | 内容胶囊 |
    购买 | 出售`；不可买/卖一侧显示 `—`。
- **数据支撑**：为让前端能渲染列表/分区，给共享模型 `PlayerCard` 新增可选 `Meta`
  字典（`kind`/`itemId`/`buy`/`buyWas`/`sell`/`qty`，价格为已格式化字符串）；`ShopPlayerMenu` 填充；
  `WebPanelHttpServer.AppendPlayerView` 序列化 `meta` 对象；通用渲染器忽略不认识的键，**其他玩家菜单
  不受影响**。

## 为什么

- 截图显示 Medkit 卡片与高一截的礼包卡片并排被拉到等高，中间一大块空白——根因是单网格 + 跨类等高。
  分区 + 内容自适应高度即解决；列表模式进一步压缩留白，贴合窄屏。

## 影响的文件 / 范围

- `src/Shared/UnturnedMods.Shared/WebPanel/PlayerMenu.cs`（`PlayerCard.Meta`）。
- `well404.Shop/ShopPlayerMenu.cs`（填充 meta：kind/itemId/buy/buyWas/sell/qty）。
- `well404.WebPanel/WebPanelHttpServer.cs`（`AppendStringMap` + 在卡片 JSON 里输出 `meta`）。
- `well404.WebPanel/player.html`（shop 专属 `renderShop`/`shopSection`/`buildShopTile`/`buildShopRow`/
  `priceButton`；新增 `.tilegrid`/`.shoplist`/`.srow`/`.modeToggle` 等 CSS；STR 加 items/bundles/
  cardMode/listMode）。
- 文档：`docs/well404.Shop.md`（玩家商店两种视图 + 分区说明）。

## 注意事项 / 后续

- **已用无头 Chrome（google-chrome --headless=new）经 dev-player 会话实测四张截图**：卡片/列表 ×
  宽屏(1280)/窄屏(430) 均正确——单品行 `#15 Medkit 购买$100 出售$40`、礼包行 `Starter Pack
  [Medkit×2][MRE] 购买$500 —`、分区与计数、绿色买价、余额徽章、切换记忆都符合预期。
  列表视图需 `localStorage` 故用 Chrome DevTools Protocol（`cdpshot.py`，纯 stdlib 最小 ws 客户端）
  设 `shop_mode=list` 后截图。该脚本在 `$CLAUDE_JOB_DIR/tmp`，仅调试用、未入库。
- 环境坑：headless Chrome 需 `--user-data-dir` 指向可写目录且 `HOME`/`TMPDIR` 也指过去（本机
  `/tmp` 权限受限），dbus/ProcessSingleton 报错可忽略。
- 经验：玩家展示面优先**按品类分区 + 内容自适应**，避免单网格等高拉伸留白；面向 Steam 内嵌浏览器
  要给「紧凑列表」选项。复杂结构化展示用 `PlayerCard.Meta` 传结构化数据，别让前端去解析本地化字符串。
