# 配置 NuGet 与 GitHub 发布、CI，并将许可改为 CC BY-NC-SA 4.0

## 做了什么
- **GitHub Actions**：
  - `.github/workflows/ci.yml`：push / PR 到 `main` 时 restore → build → test。
  - `.github/workflows/publish.yml`：发布 GitHub Release 时 restore → build →
    `dotnet pack --no-build` → `dotnet nuget push --skip-duplicate`（用 secret
    `NUGET_API_KEY`）。
- **包元数据**（`Directory.Build.props`）：`RepositoryUrl`/`PackageProjectUrl`
  指向 `Well2333/unturned-mods`；打包 `README.md` 与 `LICENSE`；启用符号包
  （`.snupkg`）与 SourceLink（新增 `Microsoft.SourceLink.GitHub` 到 CPM）。
- **许可变更**：MIT → **CC BY-NC-SA 4.0**。`LICENSE` 换为官方法律文本；因 CC 非
  OSI/FSF 许可，NuGet `PackageLicenseExpression` 不接受，改用
  `PackageLicenseFile=LICENSE` 捆绑发布。
- **修复插件模板**：`Plugin.cs.template` 缺少 `using OpenMod.API.Plugins;`，导致
  `PluginMetadata` 编译失败（CS0246）；已补上。
- **重建解决方案**：手写的 `.sln` 含畸形的解决方案文件夹条目，`dotnet sln` 报
  MSB5006；改用 `dotnet new sln` 重新生成干净的 `UnturnedMods.sln`。
- 新增一级规范 `memory/guidelines/publishing.md` 并登记进 `00-index.md`。

## 为什么
- 用户要求把插件发布到 NuGet 与 GitHub；随后要求许可改为 CC BY-NC-SA 4.0。

## 影响的文件 / 范围
- `Directory.Build.props`、`Directory.Packages.props`、`UnturnedMods.sln`、
  `LICENSE`、`build/templates/plugin/Plugin.cs.template`、
  `.github/workflows/{ci,publish}.yml`、`memory/guidelines/{publishing,00-index}.md`。

## 注意事项 / 后续
- **必须在仓库设置中添加 secret `NUGET_API_KEY`** 才能发布：
  `gh secret set NUGET_API_KEY --repo Well2333/unturned-mods`（密钥勿入库/对话）。
- **发布前需递增对应插件 `.csproj` 的 `<Version>`**，否则 `--skip-duplicate` 会跳过。
- 已用临时插件端到端验证：build 0 错误，pack 产出 `.nupkg`+`.snupkg`，包内含
  `LICENSE`/`README.md`，`config.yaml`/`translations.yaml` 正确嵌入。SourceLink 在
  无远程时会告警，推送到 GitHub 后消失。
- 已知：`UnturnedMods.Shared` 不单独发布；若插件引用它并打包，需把其 dll 嵌入插件
  包（见 publishing.md「已知注意事项」）。
