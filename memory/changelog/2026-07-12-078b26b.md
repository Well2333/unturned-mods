# 修复 reload 后业务插件指令与 WebPanel 扩展消失

## 做了什么
- 六个向 WebPanel 提供功能的插件统一增加幂等注册入口：插件自身加载时注册，并在后续
  `PluginLoadedEvent` 上重试尚未完成的可选注册，覆盖 WebPanel/全局服务晚到的加载顺序。
- 卸载时只从加载期保存的 registry 实例注销，不再从可能已释放的插件作用域解析服务；
  AutoSave 同步保存晚注册所需的 BackupService。
- 明确二进制更新边界：安装、升级或覆盖插件 DLL / `UnturnedMods.Shared.dll` 后必须完整重启
  服务器，`openmod reload` 仅用于未替换 DLL 时的配置和同版本生命周期重载。
- 构建脚本、根文档、部署文档与架构/开发/本地测试规范同步更新上述规则。
- 补丁版本提升：AdminTools 1.0.1、AutoSave 1.0.1、Economy 1.0.2、Essentials 1.0.3、
  Shop 1.0.2、Vault 1.0.2。

## 为什么
- 现场在运行中更新包并执行 `openmod reload` 后，只剩 WebPanel 自带的 `/menu` 与首页；
  `/tp` 等业务命令、钱包/实用工具/商店/仓库菜单同时消失。这表明不是 Cloudflare 或前端
  单点故障，而是业务插件/命令权限源和跨插件注册处于不完整的混合加载状态。
- Mono/OpenMod 软 reload 不能可靠卸载已经载入的程序集。运行中替换不同版本 DLL，尤其多个
  包都内嵌的 Shared DLL，可能让新旧类型身份并存；代码只能自愈正常加载顺序，二进制升级仍
  必须完整重启。

## 验证
- `scripts/build.sh --test`：80 项单元测试全部通过，七插件 Release 构建成功。
- 本地 Test 服完整部署后执行 `openmod reload`：七插件全部干净卸载/加载，无
  `ObjectDisposedException`；注册日志完整。
- reload 后通过 dev-player token 请求 `/api/p/view`，确认返回
  home/economy/essentials/shop/vault，首页包含“实用工具”以及 `/tp`、`/warp` 等说明。
- Cloudflare Quick Tunnel 在 reload 后重新取得公网 URL；关服后 U3DS/cloudflared 无残留。

## 部署注意
- 不要在服务器运行时覆盖这批 DLL 后只执行 `openmod reload`。
- 安装全部新版本后完整停服并启动；随后由在线玩家验收 `/tp`、`/warp 绿洲` 与 `/menu` 标签。
