# 玩家面板:用「页内输入框」取代 window.prompt(Steam 内嵌浏览器会忽略原生弹窗)

用户在线打开仓库后问「数量在哪输入?」——点「数量」毫无反应。

## 根因

仓库「数量」(以及商店「出售数量」)用的是 `window.prompt` 询问数字。**Steam 游戏内嵌浏览器
会静默忽略 `window.prompt()`**(及 alert/confirm),所以点击后什么都不弹,玩家根本没地方输入。
我之前用普通 Chrome 经 CDP 测试时 `window.prompt` 正常,因此没复现到。

## 做了什么

- `player.html` 新增 `askValue(label, default)`:一个**页内浮层**(`.pmodal-overlay`/`.pmodal`),含
  聚焦的 `<input type=number>` + 确定/取消;回车提交、Esc/点背景取消;返回 Promise(取消为 null)。
- `invoke()` 把 `window.prompt(...)` 换成 `await askValue(...)`。**纯 DOM**,在 Steam 内嵌浏览器里也能渲染。
- `promptDefault` 约定不变:`""` → 开空白框(仓库数量,标题带 `(1–N)`);缺省 → 回退 `1`(商店出售数量)。
- 顺带补 `confirm`/`cancel` 两个前端 i18n 串 + `.pmodal*` 样式 + `button.act.ghost`。

## 影响的文件 / 范围

- WebPanel:`player.html`(`askValue` + 样式 + i18n + `invoke` 改用页内框)。**通用渲染器**,
  所有带 `promptLabel` 的玩家按钮(仓库数量、商店出售数量)都受益。
- 仅改 WebPanel.dll;`player.html` 为内嵌资源,重新部署 WebPanel.dll 即可。

## 验证 / 注意

- **CDP 截图确认**:点「数量」弹出页内框,标题 `取出数量 (1–3)`,内含空白数字输入框 + 取消/确定。
- 教训:**面向 Steam 内嵌浏览器的功能,不能用 `window.prompt/alert/confirm`**,要用页内元素;用普通
  浏览器测会漏掉这个差异。后续任何玩家面板交互都遵此。
- 实际「取出 X 个」的搬运是在线操作,走的是已实测的 take_some + 变体过滤路径(本次只改了输入 UI)。
