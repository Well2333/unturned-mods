# 加固本地构建的版本打标(避免旧 obj 残留导致 dll 版本号过期)

## 做了什么
- `scripts/build.sh` 在构建前删除 `src/**/obj/$CONFIG/**/*.AssemblyInfoInputs.cache`,
  强制按当前 `<Version>` + git HEAD 重新生成程序集的 `AssemblyVersion` /
  `InformationalVersion`。借 `WriteOnlyWhenDifferent`,仅在真的过期时才改写并重编。
- 在 `development-standards.md` §5 加了对应说明。

## 为什么
- 本地增量构建若复用了上一个版本遗留的 `obj/`,可能把**过期版本号**打进部署的 dll,
  使 OpenMod `[loading] <name> v<version>` 日志显示错误版本(NuGet 包版本不受影响,
  CI 全新 checkout 也无此问题)。本轮排查 `openmod install` 时正好踩到这个误导现象。

## 影响的文件 / 范围
- `scripts/build.sh`、`memory/guidelines/development-standards.md`。
- 仅影响本地 `build.sh` 部署;不改任何插件运行代码,**不发布新版本**。

## 注意事项 / 后续
- 已验证:人为把某插件 obj 的信息版本改成 `9.9.9+stale`,仅删该 cache 后重建即被纠正回
  `<Version>+HEAD`。
- 线上已发布的 0.2.2 / 0.1.2 包经 nuget.org 下载核对,DLL 信息版本本就正确
  (`0.2.2+<sha>`),无需重发。

## 规范同步(guideline_changed = true)
- `development-standards.md` §5 新增「版本打标」说明。
