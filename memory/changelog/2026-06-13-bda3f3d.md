# 端口默认改 27020 + 两处上线后测试发现的 bug 修复

## 做了什么
- **端口默认 8080 → 27020**(前置提交 `6b9c815`):8080 常被其他 Web 服务占用;改到 Unturned
  服务器端口(27015–27017)附近且避开它们,便于一起转发。改了 `WebPanelSettings.cs` 默认值、
  嵌入 `config.yaml`、相关文档;测试服的部署配置也同步改为 27020(实测仅 27020 监听,8080 已无)。
- **修复①:介绍页崩溃**(`本次 bda3f3d`)。`IPlayerCommandRegistry` 登记的指令权限串写错——
  Essentials 漏了 `commands.` 段(`well404.Essentials:home` 应为 `well404.Essentials:commands.home`),
  Economy/Shop/WebPanel 组件 id 大小写错(`economy`→`Economy` 等);而 OpenMod 对**未注册的权限**
  会**抛异常**,导致整个介绍页 `RenderAsync` 抛错。已把全部权限串按 `ICommandPermissionBuilder`
  的真实输出 `<PluginId>:commands.<cmd>` 改正,并把 `IntroPlayerMenu` 的权限判断**做成不抛**
  (`CanUseAsync` 包 try/catch,解析不了的权限默认「显示该指令」,绝不再让整页崩溃)。
- **修复②:AdminTools 权限组指令报「IPermissionStore has not been registered」**。本 OpenMod 宿主
  未注册 `IPermissionStore`;改用已注册的 `IPermissionRolesDataStore`(读 `PermissionRoleData`,
  增删其 `Permissions` 后 `SaveChangesAsync`)。服务器实测:查看/授予/撤销 + 持久化均通过。

## 为什么
- 端口:用户要求避开 8080、靠近 Unturned 端口。
- 两处 bug 均在用户上服测试时暴露,属上线即修。

## 影响的文件 / 范围
- 端口:`WebPanelSettings.cs`、`config.yaml`、`docs/well404.WebPanel.md`、`README.md`(WebPanel)。
- 修复①:`IntroPlayerMenu.cs`(crash-safe 判断)、`EconomyPlugin.cs`/`ShopPlugin.cs`/
  `WebPanelPlugin.cs`/`EssentialsPlugin.cs`(权限串改正)。
- 修复②:`well404.AdminTools/AdminToolsService.cs`(`IPermissionRolesDataStore`)。

## 注意事项 / 后续
- 经验:**OpenMod 指令权限节点是 `<PluginId>:commands.<cmd>`(组件 id 区分大小写,即 PluginMetadata
  里的名字);`CheckPermissionAsync` 对未注册权限会抛异常**,凡是要校验外部权限串的地方都应
  try/catch 兜底。`IPermissionStore` 在本宿主不可解析,角色权限的增删用 `IPermissionRolesDataStore`。
- 已在测试服重启验证;服务器保持运行供用户继续在游戏内 `/menu` 联调(隧道每次重启换新随机域名)。
