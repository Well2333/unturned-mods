# Shop 玩家 WebUI 新增一键售卖

## 做了什么

- 玩家网页商店顶部新增「快捷操作 / 一键售卖」按钮，无需输入数量即可执行。
- 扫描玩家背包并回收商店目录中所有 `sellPrice > 0` 的普通商品，按各商品卖价汇总后一次入账。
- 明确排除礼包，避免礼包内容与普通商品或其他礼包重叠时因配置顺序产生歧义。
- 增加中英文界面提示、空背包/无可售目录反馈，并把 Shop 版本提高到 1.0.5。
- 更新插件 README 与详细文档，说明操作范围和无确认行为。

## 为什么

- 玩家原先只能逐个商品输入出售数量，清理大量可回收物品操作繁琐。
- 使用通用 PlayerMenu 按钮协议即可实现该操作，无需更新 WebPanel 宿主版本。

## 影响的文件 / 范围

- `src/plugins/well404.Shop`
- `tests/well404.Shop.Tests`
- `docs/well404.Shop.md`

## 注意事项 / 后续

- 一键售卖只处理普通商品；礼包继续通过对应商品行手动出售。
- 按钮是无数量输入、无二次确认的即时操作，界面以红色危险操作样式提示。
- Shop 13 项测试通过，覆盖按钮动作/无提示框、目录筛选和多商品金额汇总。
- 全仓 Release 测试共 106 项通过；Shop Release 构建 0 警告、0 错误，并生成 1.0.5 NuGet 包。
- 背包实际扣物依赖 Unturned 运行时，本地无在线游戏玩家环境，按测试规范以纯逻辑测试和编译检查验证。
