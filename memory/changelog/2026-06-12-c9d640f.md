# 修复 `openmod install` 解析失败 + 每包独立 README + Shop 硬依赖 Economy

## 做了什么
- **修复幽灵依赖（核心 bug）**：三插件原本把内部库 `UnturnedMods.Shared` 写进了 nuspec
  依赖，而它从未发布到 NuGet，导致 `openmod install` 抛
  `NuGetResolverConstraintException`。改为 `ProjectReference ... PrivateAssets="all"`：
  既不进依赖列表、又仍由 `IncludeSharedInPackage` 把 `UnturnedMods.Shared.dll` 打进
  `lib/`。插件脚手架模板（`build/templates/plugin/Plugin.csproj.template`）同步修正。
- **Shop 硬依赖 Economy**：Shop 通过对可打包项目 `well404.Economy` 的普通
  `ProjectReference`，让 nuspec 自动生成 `<dependency id="well404.Economy">`，
  `openmod install well404.Shop` 会级联安装 Economy。
- **每包发布自身 README**：`Directory.Build.props` 改为优先把插件自带 `README.md` 选为
  `PackageReadmeFile`（缺则回退仓库根 README）；本地 pack 与 CI 一致。新增三份插件 README
  （含「插件家族」交叉推广），`new-plugin.sh` + 新模板会给新插件生成 README 骨架。
- **文档刷新**：根 README / `docs/` 补 WebPanel、修正 Shop 依赖说明、新增
  `docs/well404.WebPanel.md`。
- **版本**：Economy 0.2.2、Shop 0.2.2、WebPanel 0.1.2。

## 为什么
- 用户服务器 `openmod install well404.Shop` 失败（先是缺 `allowedActors` 的配置报错，
  补配置后即暴露出这个打包层的依赖解析失败）。根因是打包配置，不是运行代码。

## 影响的文件 / 范围
- 三插件 `.csproj`、`Directory.Build.props`、`build/templates/plugin/*`、`scripts/new-plugin.sh`。
- 文档：根 `README.md`、`docs/README.md`、`docs/well404.{Economy,Shop,WebPanel}.md`、
  三插件 `README.md`。
- 规范：`development-standards.md`、`commit-and-memory-workflow.md`、`CLAUDE.md`。
