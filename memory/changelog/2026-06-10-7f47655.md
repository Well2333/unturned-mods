# 真实服务器实测发现并修复的 DI/部署问题

在本地 U3DS + OpenMod 3.8.10 无头服务器上实测，发现并修复了三处编译期/单测都抓不到
的问题。

## 做了什么 / 为什么

1. **EconomyProvider 无法激活**（`None of the constructors can be invoked`）。
   - 全局 `[ServiceImplementation(Singleton)]` 在**全局作用域**激活，那里不存在插件
     作用域的 `IConfiguration` / `IOpenModComponent`，故构造失败。
   - 修复：改回注入 **`IPluginAccessor<EconomyPlugin>`** + `IUserManager`，在调用时懒读
     `Instance.LifetimeScope.Resolve<IConfiguration>()` 与 `Instance.WorkingDirectory`。
2. **Shop 的 `ShopCatalog`/`DiscountService`/`ShopService` 未注册**
   （`Unable to resolve service …`）。
   - 这些具体类没有 `[Service]` 接口，`[PluginServiceImplementation]` 只会警告
     “does not inherit any services” 并跳过。
   - 修复：去掉这些属性，新增 `ShopContainerConfigurator : IPluginContainerConfigurator`，
     用 `RegisterType<T>().AsSelf().SingleInstance()` 注册。
3. **`build.sh` 部署需平铺**。OpenMod `FileSystemPluginAssembliesSource` 只扫描
   `plugins/*.dll` 顶层、不递归子目录，之前按 `build/<PluginId>/` 子目录部署导致
   “0 plugins loaded”。改为**平铺**输出（`build/*.dll`），并相应更新 docs/规范。

## 实测验证（控制台 + 日志，全自动）
- 两插件加载无 warning；`> 2 plugins loaded`。
- `balance` → `$0`；`eco give 500` → `$500`；`eco take 200` → `$300`；
  `eco set 1000` → `$1000`；`shop` 正确列出 medkit/starter。
- LiteDB 写入磁盘（`plugins/well404.Economy/economy.db`），**服务器重启后余额仍为
  $1000**（持久化 OK）。
