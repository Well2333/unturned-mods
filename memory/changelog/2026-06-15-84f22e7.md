# 1.0.0 全插件协调发布(版本号 bump)

用户明确授权:清理垃圾文件后,提交并发布 **1.0.0 正式版(全部插件)**。本提交只做版本号 bump。

## 做了什么

- 6 个插件 `<Version>` 统一升到 **1.0.0**(经 `scripts/plugin-version.sh`):
  - Economy 0.4.0 → 1.0.0
  - Shop 0.4.0 → 1.0.0
  - WebPanel 0.4.0 → 1.0.0
  - Essentials 0.3.0 → 1.0.0
  - AdminTools 0.1.0 → 1.0.0
  - Vault 0.1.0 → 1.0.0

## 发布方式

- `publish.yml` 支持**协调式整库发布**:发布一个 tag **不**匹配 `<PluginId>/v<Version>` 模式的 GitHub Release
  (即 `v1.0.0`)→ `mode=all` → `dotnet pack` 整个解决方案 → `dotnet nuget push --skip-duplicate` 推送所有插件。
  NuGet 认证走 **OIDC Trusted Publishing**(无需长期 API key)。
- 因此本次用**单个 `v1.0.0` Release**(targeting main)一次性发布全部 6 个插件,而非 6 个 per-plugin tag。

## 验证 / 注意

- `dotnet pack UnturnedMods.sln -c Release`:6 个插件均产出 `*.1.0.0.nupkg`,**0 警告 0 错误**。
- 分支整合:本功能分支 `fix/openmod-install-packaging` 内容是 `origin/main`(含 PR #1 打包修复)的**严格超集**
  (逐文件核对:csproj 仅版本号不同、无 main-only 文件),故用 `merge -s ours origin/main` 吸收分叉提交后并入 main,
  不丢失任何内容。
- `--skip-duplicate`:已存在的版本不会重推;6 个 1.0.0 均为新版本,故全部推送。
