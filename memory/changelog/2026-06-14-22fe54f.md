# 新插件 well404.Vault：私人仓库（背包物品存取，完整保真，按格子计容量）

用户要求新建「私人仓库」插件：通过指令或 Web 面板存入/取出背包中的物品;容量优先按物品占用的背包
格子数计(如弹药箱 2×2=4 格),并注意弹匣/弹药箱这类「单物品内有堆叠数」的占用问题(39 发的弹药箱不能
算 39 个);默认 200 格。两个设计点经询问确认:**按格子(默认200)** + **完整保真**。

## 做了什么

- 脚手架 `scripts/new-plugin.sh well404.Vault` 生成,启用 Shared 项目引用。
- **完整保真存储**:`StoredItem` 存 id+amount+quality+state(原始状态字节,base64),取出用
  `new Item(id,amount,quality,state)` 原样重建——枪械配件、弹匣/弹药箱内弹药、耐久都保留。
  数据经 OpenMod `IDataStore` 持久化到 `vault.data.yaml`(按 Steam ID 分组)。
- **按格子计容量**:`SlotCostOf(itemId) = max(1, asset.size_x*asset.size_y)`;`UsedSlots` = 已存物品
  占格之和;`maxSlots` 默认 200。**关键**:占用按「每个物品实例的网格尺寸」算,**不乘** item.amount、
  也不看内部弹药数——所以 39 发弹药箱只占其格子尺寸,不被当成 39 个。读不到尺寸退化为 1 格。
- **库存操作走 SDG.Unturned**(`user.Player.Player.inventory`):遍历 `PlayerInventory.SLOTS..STORAGE`
  各页(跳过装备槽与打开的外部存储);存入 `getItem`/`getIndex`/`removeItem`;取出 `tryAddItem(item,true)`,
  背包满则 `ItemManager.dropItem` 掉在脚下。全部 `await UniTask.SwitchToMainThread()`。
- **指令**(无硬依赖):`/vault`(用法)、`/vault store|deposit <物品ID> [数量]`、
  `/vault take|withdraw <物品ID> [数量]`、`/vault list|info`;均按游戏物品 ID 操作。
- **可选 Web 面板**:玩家「仓库」菜单(背包区 Store 按钮 + 仓库区 Take 按钮 + 容量徽章,经**通用**玩家
  菜单模型 Group/Badge/Layout/Buttons 渲染,宿主零仓库逻辑);管理面「仓库」设置模块编辑 `maxSlots`
  (满足 §8 配置须 WebUI 可编辑)。`maxSlots` 经 `VaultConfigStore` 重写 config.yaml。

## 为什么

- 用户需求。按格子+完整保真是用户两次选择「推荐」项;SDG 直接操作库存是唯一能拿到原始状态字节与
  物品网格尺寸的途径(OpenMod 抽象不暴露 raw state)。
