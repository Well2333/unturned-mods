# 玩家指令介绍双语化 + AdminTools 瘦身 + 实用工具补全 + 修复中文界面英文泄漏

用户上服测试后反馈四点（且强调这些都能在网页直接 debug、不该半成品交付）。

## 做了什么

- **①玩家介绍页指令列表双语且详细**。原先 `PlayerCommandInfo.DescriptionKey` 用的是点号键
  （`essentials.cmd.home`），且只在各插件 ZhTable 里有中文；英文（默认）面板因此直接显示
  原始键，中文也过于简短。改为与面板其余部分一致的约定（**英文源串即 i18n 键**）：每条指令
  现在带详细英文描述 + 分组标题，中文在各插件 `WebI18n.ZhTable` 里给全。Essentials 还补了
  `/home set`。涉及 Economy/Shop/Essentials/WebPanel 四个插件的注册 + 各自 ZhTable。
- **②移除 AdminTools 的权限组/指令授予 Web 功能**（用户认为做得差）：删掉权限组表、玩家权限组
  增删、查找指令、权限组指令授予/撤销、查看某组已有权限。保留治安功能：在线玩家、godmode、
  踢出、封禁、解封。`AdminToolsService` 同步瘦身（去掉 `IPermissionRoleStore`/
  `IPermissionRolesDataStore`/`ICommandStore` 等依赖与 `PlayerPermissionActor`）。
  `AdminResult` 改为「英文模板 + 参数」，面板按语言本地化结果消息，不再发原始英文。
- **③补全玩家「实用工具」菜单**：新增「把家设在这里」(`/home set`)；队长可对每名队员「踢出队伍」
  （`PartyMember` 增加 `SteamId`，`PartyService` 增加 `IsLeader`）。
- **④修复中文界面的英文泄漏**：结果表的列头与结果消息此前原样输出，中文面板会冒出英文
  （“Granted permission”“Name”、godmode 确认语等）。现在 `BuildResultJson` 用翻译注册表本地化
  **消息 + 列头**；补齐列头与 Essentials 残余字符串的中文；修正 `index.html` 里硬编码的「行」
  计数改为随界面语言。

## 为什么

- 均为用户上服测试发现的质量问题，要求认真修复、勿再半成品交付。

## 影响的文件 / 范围

- 介绍页：`EssentialsPlugin.cs`/`EconomyPlugin.cs`/`ShopPlugin.cs`/`WebPanelPlugin.cs` 的注册 +
  四个 `WebI18n.cs`/`WebPanelI18n.cs`。
- AdminTools：`AdminToolsWebPanelModule.cs`、`AdminToolsService.cs`（含 `AdminResult` 重构）、
  `AdminToolsPlugin.cs`、`WebI18n.cs`；删除 `PlayerPermissionActor.cs`。
- 实用工具：`EssentialsPlayerMenu.cs`、`Party/PartyService.cs`、`Essentials/WebI18n.cs`。
- i18n 泄漏：`WebPanelHttpServer.cs`（`BuildResultJson` 本地化消息+列头）、`index.html`。
- 文档：`docs/well404.AdminTools.md`、`docs/README.md`、`README.en.md`、
  `src/plugins/well404.AdminTools/README.md`、`docs/well404.Essentials.md`、
  `src/plugins/well404.Essentials/README.md`。

## 注意事项 / 后续

- 服务器实测（端口 27020）：5 插件全部加载并注册；ZH 面板逐项扫描——模块标题/动作标签/字段/
  占位符/列头/结果消息全部中文，仅余**有意保留的技术标识**（`SteamID` 以及 Economy 后端枚举值
  `database`/`experience`，因其为提交给配置的实际取值，翻译会破坏写入）。AdminTools 现仅
  players/godmode/kick/ban/unban。
- **玩家介绍页/实用工具**属玩家面（需玩家在游戏内 `/menu` 生成会话），无法用 curl 验证；其文本走
  与已验证的管理面**同一套翻译注册表**，因此中英解析逻辑已被间接证明。仍需用户在游戏内点验
  「把家设在这里」「队长踢人」等交互。
- 经验：面板列头/结果消息也是 i18n 键，新增表格/结果时记得在 ZhTable 配中文，否则中文面板泄漏英文。
  指令帮助统一用「英文源串即键」，**不要再用点号键**，否则英文面板会显示原始键。
