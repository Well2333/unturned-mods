# WebPanel 找不到 cloudflared 时自动下载便携版 + 修复服务器关停时的 ObjectDisposedException

## 做了什么
- **自动下载便携版 cloudflared**(well404.WebPanel,1.0.0 → **1.1.0**):
  - 新增 `CloudflaredDownloader`,在 `type: cloudflare` 且 `command`(默认 `cloudflared`)
    在磁盘与 `PATH` 中都找不到时,从 Cloudflare 官方 GitHub 下载**最新版**便携二进制到
    插件数据目录的 `cloudflared/` 子目录并运行它——**绝不**安装到系统、不写入 `PATH`。
  - 下载结果**缓存复用**(文件已存在且非空则不再下载);Linux 自动 `chmod +x`;
    通过临时文件 + 原子移动避免半截文件被复用。
  - 支持 **Windows / Linux(x64、arm64;x86、arm 也覆盖)**;macOS 等(发行包为 `.tgz`)
    不自动下载,打印提示让管理员手动安装。失败回退原 `command` 仅告警,面板本地照常可用。
  - 新增配置项 `web.tunnel.autoDownload`(默认 `true`,仅 `cloudflare` 生效;`custom` 不下载)。
- **修复关停崩溃**(5 个插件;Vault/Shop/Economy/Essentials 1.0.0 → **1.0.1**):
  `OnUnloadAsync` 不再从 `LifetimeScope` 解析服务——改为在 `OnLoadAsync` 把
  `IPlayerCommandRegistry`(WebPanel 还有 `IWebPanelRegistry`/`IPlayerMenuRegistry`)缓存到
  字段,卸载时用缓存引用做反注册。

## 为什么
- 真机部署常未预装 cloudflared,开隧道即报 `Win32Exception 系统找不到指定的文件`;自动下载便携版
  让其开箱即用且不污染系统(用户要求)。
- 服务器整体关停时 OpenMod 先释放 Autofac LifetimeScope,插件 `OnUnloadAsync` 再 `Resolve*`
  就抛 `ObjectDisposedException`(日志里 5 个插件各报一次)。注册表本是全局单例(字典对象),
  加载时缓存、卸载时复用即可根治,且不影响单插件 reload 时的正常反注册。
